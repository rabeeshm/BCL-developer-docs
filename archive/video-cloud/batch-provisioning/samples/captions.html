Adding Captions Using Batch Provisioning

Closed captions can be added to your videos to provide additional or interpretive information to individuals who wish to access it. Associating a caption file with a video can be done via the Media module or via batch provisioning. This document provides example code for seven different scenarios of associating a captioning file with a video. 

Note: this sample applies only to DFXP captions. Batch Provisioning cannot be used to add or update WebVTT captions.

Brightcove supports adding captions to your videos using the Media module in Video Cloud Studio, by using the Smart Player API or by using FTP Batch Provisioning. For information, see Using Captions with the Smart Player API or Displaying Captions for a Video.

When using batch provisioning, the only file allowed to have an .xml extension is the XML manifest file. If any other files have an .xml extension errors will result. Use the .dfxp extension for caption files.

Brightcove supports both the Society of Motion Picture and Television Engineers Timed Text Format (SMPTE-TT) and Distribution Format Exchange Profile (DFXP) formats for closed captions.

Code Examples

The following are examples of XML batch provisioning files to add captioning to videos:

Add hosted to existing

Here is a code example on how to add a hosted captions file to an existing video.

            <?xml version="1.0" encoding="utf-8" ?> 
<publisher-upload-manifest publisher-id="8523" preparer="Eamon" report-success="true">
 <notify email="batchman@brightcove.com" /> 
  <asset filename="caption68.dfxp" 
      refid="caption68" 
      type="CAPTION" />  
  <title refid="video-5"
      overlay-update="true"> 
      <caption-refid>caption68</caption-refid>    
 </title>  
</publisher-upload-manifest>

Add remote to existing

Here is a code example on how to add a remote captions file to an existing video.

            <?xml version="1.0" encoding="utf-8" ?> 
<publisher-upload-manifest publisher-id="8523" preparer="Eamon">
  <notify email="batchman@brightcove.com" /> 
  <remote-asset display-name="caption file" 
      refid="caption78"
      remote-url="http://www.test.com/caption.dfxp"
      size="1470" 
      type="CAPTION" />  
  <title refid="video-7" 
      overlay-update="true"> 
      <caption-refid>caption78</caption-refid>    
 </title>  
</publisher-upload-manifest>

Here is a code example on how to add a remote captions file to a remote video.

Create video with hosted

Here is a code example on how to create a video with a hosted captions file.

            <?xml version="1.0" encoding="utf-8" ?> 
<publisher-upload-manifest publisher-id="8523" preparer="Eamon">
  <notify email="batchman@brightcove.com" /> 
  <asset filename="caption41.dfxp" 
      refid="caption41" 
      type="CAPTION" />   
  <asset filename="galactica_preview.flv" 
      refid="captionVideo78" 
      type="VIDEO_FULL" /> 
  <title name="Captioned Video 41" 
      refid="video-97" 
      active="TRUE" 
      video-full-refid="captionVideo78"> 
    <caption-refid>caption41</caption-refid> 
    <short-description>non-remote video with non-remote captions</short-description>     
 </title>
</publisher-upload-manifest>

Create remote with hosted

Here is a code example on how to create a remote video with a hosted captions.

            <?xml version="1.0" encoding="utf-8" ?> 
<publisher-upload-manifest publisher-id="8523" preparer="eamon">
  <notify email="batchman@brightcove.com" /> 
  <asset filename="caption35.dfxp" 
      refid="caption35" 
      type="CAPTION" />  
  <remote-asset display-name="remote caption video" 
      refid="remoteVideo22"    
      remote-url="rtmp://cp87609.edgefcs.net/ondemand/"
      remote-stream-name="media/doNotDelete/remoteAssets/lebron"
      size="1344444"
     video-codec="SORENSON"
      video-duration="25000" 
      type="VIDEO_FULL" /> 
  <title name="Remote Lebron" 
      refid="Lebron" 
      active="TRUE" 
      video-full-refid="remoteVideo22">  
    <caption-refid>caption35</caption-refid> 
   <short-description>A remote video that has non-remote captions</short-description>     
 </title>  
</publisher-upload-manifest>

For further help

If you need help getting your Batch FTP manifest file working correctly, Brightcove Support is available to help. You can submit a case here. To make sure you get the fastest response possible, below is a list of what support will need to solve the problem.

  • Attach the original XML manifest file that you submitted
  • The success/failure message from the batch system that was generated and sent to you (if any)
    • Please supply the full text from the email
    • Please refrain from further submitting the manifest until support has had a chance to diagnose the issue
  • Confirm if this is an automated upload system on your end or if the files are being uploaded manually via FTP software

After your case is submitted, you will receive an email confirmation from Brightcove Support. To provide additional information on your case to Brightcove Support, reply to the confirmation email.
