Overview

This version of the Analytics API is obsolete. Please see documentation for the current version.

The referrer_domain dimension is the top private domain of the referrer URL on a page where video events are collected (this URL is sent to Collector with the referrer parameter). The referrer_domain dimension only includes metrics from player instrumentation events.

referrer_domain in the API

This version of the Analytics API is obsolete. Please see documentation for the current version.
Referrer URL API Value
http://blog.brightcove.com/en/ (or any valid URL) brightcove.com (top private domain for the URL)
(invalid or missing url) direct

Metrics

The following metrics are available for the referrer_domain dimension:

  • player_load
  • video_impression
  • video_view

Metadata

There is no metadata collected for a referrer_domain.

Rollup Data

Rollup data can be requested for a specific referrer_domain.

Parameters

Parameter Supported
where referrer_domain (required)
from yes
to yes
limit yes
offset yes
sort yes

Example

Request: https://data.brightcove.com/analytics-api/videocloud/accounts/1234?where=referrer_domain==brightcove.com

Report Data

Report data for an account can be broken down by referrer_domain.

The where parameter is supported for the player dimension for referrer_domain (where is not yet supported for the video dimension).

Parameters

Parameter Supported
dimensions referrer_domain
where player
from yes
to yes
limit yes
offset yes
sort yes

Example

Request: https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report?dimensions=referrer_domain&limit=3&sort=video_view

Response:

{
        "account": "1234",
        "item_count": 7,
        "items": [
            {
                "referrer_domain": "brightcove.com",
                "player_load": 193,
                "video_impression": 150,
                "video_view": 120
            },
            {
                "referrer_domain": "clubbingabroad.co.uk",

            },
            {

Request: https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report?dimensions=referrer_domain&limit=3&sort=video_view&where=player==567

Functional Expectations

Given that [video_impression, video_view] events have been collected for referrer http://blog.brightcove.com/en
        And I make a call for report data with dimensions=referrer_domain
        Then I get a report that includes "referrer_domain": "brightcove.com"
        And I make a call for rollup data where=referrer_domain==brightcove.com
        Then I get a rollup report that includes trending data for video_impression and video_view
    Given that [video_impression, video_view] events have been collected with no specified referrer
        And I make a call for report data with dimensions=referrer_domain
        Then I get a report that includes "referrer_domain": "direct"
        And I make a call for rollup data where=referrer_domain==direct
        Then I get a rollup report that includes trending data for video_impression and video_view
    Given that [video_view, video_impression] events have been collected for referrer " "
        And I make a call for report data with dimensions=referrer_domain
        Then I get a report that includes "referrer_domain": "direct"
    Given that [video_impression, video_view] events have been collected for multiple referrers
        And I make a call for report data with dimensions=referrer_domain
        Then I get a report with referrer_domains with video_impression and video_view summary data
    Given that [video_impression, video_view] events have been collected for referrers with player 567
        And I make a call for report data with dimensions=referrer_domain&where=player==567
        Then I get a report with referrer_domains with video_impression and video_view summary data for player 567
