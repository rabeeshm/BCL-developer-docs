Dimensions and Metrics

This document is an overview of the dimensions collected for the videocloud domain, and the metrics exposed by the API. More information for a particular dimension can be found in the documentation for that dimension.


Dimensions: videocloud

The videocloud domain collects data for the following dimensions:

Dimension

Explanation

account

passed as account to the Data Collection API (should be a Video Cloud account id).

player

passed as player to the Data Collection API (should be a Video Cloud player id in the specified account).

video

passed as video to the Data Collection API (should be a Video Cloud video id in the specified account).

referrer_domain

Top private domain derived from the source parameter passed to the Data Collection API (source should be the referrer for the page where events are collected).

source_type

Source type that resulted in traffic. Derived from the source parameter passed to the Data Collection API (source should be the referrer for the page where events are collected).

search_terms

Search terms that resulted in traffic. Derived from the source parameter passed to the Data Collection API (source should be the referrer for the page where events are collected).

device_type

device types for the user's device. Derived from the user-agent or a parameter passed to the Data Collection API.

device_os

device operating system for the device of the user. Derived from the user-agent or a parameter passed to the Data Collection API.

country

Country of the user's location. Derived from the ip address, remotr_ip parameter or country parameter passed to the Data Collection API. This is a valid ISO-3601 Alpha2 country code. e.g. US for United States

region

Region/State/Province of the user's location. Derived from the ip address, remotr_ip parameter or regionparameter passed to the Data Collection API. This is a valid ISO-3601 Alpha2 region code. e.g. US-TX for Texas, United States

city

City of the user's location. Derived from the ip address, remotr_ip parameter or city parameter passed to the Data Collection API. e.g. Austin for Austin (no ISO standard codes exist at the city level)

dma

DMA code of the user's location. Derived from the ip address, remotr_ip parameter or dma parameter passed to the Data Collection API. Refer to this page for a list of dma codes.
