Overview

Destination data provides insight into where videos are being watched, i.e. the destination of traffic for video playback. (This has been called "embeds" in the past.)



Destination data in the API

Destination data consists of two pieces: destination_domain and destination_path, derived from the URI provided to Collector via the destination parameter.


Deriving destination_domain and destination_path

For reference, the generic syntax for a URI is:

<scheme name> : <hierarchical part> [ ? <query> ] [ # <fragment> ]

ex. https://www.example.com/foo/bar/baz

            \-------------/\----------/

                 |             |

             authority        path

    \---/   \------------------------/

      |                |

    scheme       hierarchical part

If the destination URI sent to Collector has an authority, the authority will be the destination_domain for that detination, and any path provided will be the destination_path in API responses. If the URI does not contain an authority, the destination_domain is the first part of the path, and the destination_path is anything remaining. A destination without a hierarchical part (e.g. just a scheme) is considered invalid, as is any value without a scheme.

For web-based playback, the destination is a URL and has an authority, and so the destination data will be handled as follows:

destination= (sent to Collector)

destination_domain

destination_path

https://www.brightcove.com/en/customers/puma

"www.brightcove.com"

"en/customers/puma"

http://www.nytimes.com

"www.nytimes.com"

""

http://some.example.com/samples?foo=bar

"some.example.com"

"samples"

For native playback (i.e. playback where there is no URL), destination data derives from the URI that identifies where the video was playing back, and will be parsed according to the rules above:

destination= (sent to Analytics)

destination_domain

destination_path

{scheme}://some.authority/roads/less-traveled

some.authority

roads/less-traveled

{other-scheme}:/Total%20Chaos/cats-and-dogs/living-together

Total%20Chaos

cats-and-dogs/living-together



Metrics

The following metrics are available for destination dimensions:

  • player_load
  • video_impression
  • video_view
  • video_percent_viewed
  • video_seconds_viewed



Metadata

There is no metadata collected for destinations.



Report Data

The Analytics API supports report data for destination_domain and destination_path (as well as destination_domain,destination_path). Using the where parameter, destination report data can be filtered by a specific video or player, or by a destination_domain (useful to see a list of destination_paths for a specific destination_domain).
