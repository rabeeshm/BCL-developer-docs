Overview

The source_type is the type of traffic source that resulted in events being collected for activity metrics. The source_type dimension only includes metrics from player instrumentation events.


source_type in the API

Sample Referrer URL

API Value

Explanation

http://www.google.com?q=brightcove

organic_search

user performed a search and clicked on a result to get to the video

http://www.google.com/aclk?q=brightcove

paid_search

user performed a search and clicked on a paid result to get to the video

http://google.com/

secure_search

user logged in with a Google account performed a search and clicked on a result (either paid or organic; search information is not included in this case)

http://blog.brightcove.com/en/

or any valid, non-search engine url

referral

user followed a link to the video

invalid or missing url

direct

user typed the link directly (or the referrer was not able to be determined)


Metrics

The following metrics are available for the source_type dimension:

  • player_load
  • video_impression
  • video_view
  • video_percent_viewed
  • video_seconds_viewed


Metadata

There is no metadata collected for source_type dimensions.

Example requests/responses for filtering source_type dimension reports by video or player are fairly redundant, and so are not included here. Note (as always) that particular summary metrics are only returned where they are configured (e.g. filtering a search_terms dimension report by a specific video does not return player_load metrics). See Dimensions and Metrics for the current supported dimension + metric combinations.


Functional Expectations

Given that [player_load, video_impression, video_view, video_engagement] events have been collected with no specified referrer

    And I make a call for report data with dimensions=source_type

    Then I get a report that includes "source_type": "direct"


    And I make a call for rollup data with where=source_type==direct

    Then I get a rollup report that includes trending data for [player_load, video_impression, video_view, video_percent_viewed, video_seconds_viewed] metrics
