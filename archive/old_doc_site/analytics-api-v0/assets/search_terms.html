Overview

The search_terms dimension is the keywords used if the traffic source that resulted in events being collected for activity metrics was a search engine, and the keywords are detectable. The search_terms dimension only includes metrics from player instrumentation events.


search_terms in the API

Search phrases are recorded largely unchanged, except:

  1. Case-sensitivity is ignored: events with searches for "brightcove", "Brightcove", and "BRIGHTCOVE" all count toward "brightcove".
  2. Leading and trailing whitespace is trimmed.
  3. Internal spaces are collapsed to a single space.
  4. Any search phrase with more than 100 characters after applying #2 and #3 is counted as "...".


Metrics

The following metrics are available for the search_terms dimension:

  • player_load
  • video_impression
  • video_view
  • video_percent_viewed
  • video_seconds_viewed


Metadata

There is no metadata collected for search_terms dimensions.

Example requests/responses for filtering search_terms dimension reports by video or player are fairly redundant, and so are not included here. Note (as always) that particular summary metrics are only returned where they are configured (e.g. filtering a source_type dimension report by a specific video does not return player_load metrics). See Dimensions and Metrics for the current supported dimension + metric combinations.


Functional Expectations

Given that [player_load, video_impression, video_view, video_engagement] events have been collected for referrer http://www.google.com?q=brightcove

    And I make a call for report data with dimensions=search_terms

    Then I get a report that includes "search_terms": "brightcove"


    And I make a call for rollup data with where=search_terms==brightcove

    Then I get a rollup report that includes trending data for [player_load, video_impression, video_view, video_percent_viewed, video_seconds_viewed] data for the search_terms "brightcove"
