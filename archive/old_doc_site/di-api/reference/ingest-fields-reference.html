Dynamic Ingest Fields Reference

This is a reference to video fields that can be used in the request body JSON for Dynamic Ingest.

Primary fields

Table 1: Primary Fields
Field Type Required Default Description
master object no none the video digital master
profile string no Video Cloud or account default profile The ingest profile to be used for transcoding
capture-images[1-1] boolean no see note[1-2] Whether poster and thumbnail images should be captured from the mid-point of the video
poster object no null Custom poster image to be ingested for the video
thumbnail object no null Custom thumbnail image to be ingested for the video
text_tracks array of text_track objects no empty array data for WebVTT captions associated with the video

Notes

  • [1-1] capture-images should be set to false if:
    • You are ingesting your own custom thumbnail and poster images, and the ingest profile has image renditions
    • This is a replace/re-transcode request and you wish to retain the current thumbnail and poster images
    • [1-2] For Dynamic Ingest without images, the default value of capture-images is true. For DI with images (poster/thumbnail), the default value of capture-images is false. If you specify it to true, system will respond 422 error.

    Also note that images will not be captured unless the profile has image renditions

Master object

Table 3: Poster Object Fields
Field Type Required Default Description
url string yes none URL for the video source; required except for re-transcoding where a digital master has been archived, or you are adding images or text tracks to an existing video

Poster object

Table 3: Poster Object Fields
Field Type Required Default Description
url string yes none URL where the image resides[3-1]
height integer no height of the source image[3-2] pixel height of the poster image
width integer no width of the source image[3-2] pixel width of the poster image

Notes

  • [2-1] Only public HTTP or HTTPS URLs or signed S3 URLs are supported
  • [3-2] For image resize, system won’t try to stretch/enlarge an image. Video Cloud will always take the unaltered requested dimensions (both width and height) as the bounding box into which we will put the output image, preserving the aspect ratio of the source, and shrinking it to fit into that box if needed.

thumbnail object

Table 4: Thumbnail Object Fields
Field Type Required Default Description

Notes

  • [4-1] Only public HTTP or HTTPS URLs or signed S3 URLs are supported
  • [4-2] For image resize, system won’t try to stretch/enlarge an image. Video Cloud will always take the unaltered requested dimensions (both width and height) as the bounding box into which we will put the output image, preserving the aspect ratio of the source, and shrinking it to fit into that box if needed.

text_track object

Table 5: Text Track Fields
Field Type Required Default Description
url string yes null URL where the .vtt file resides[5-1]
srclang ISO-639-1 language code (en, fr, ja, etc.) yes null language for the file
label string no null The label to be used in the player menu - it should be in the same language as the captions
kind enum (string) no captions how the vtt file is meant to be used - captions | subtitles | chapters | metadata
default boolean no false Setting this to true makes this the default captions file in the player menu

Notes

  • [9-1] Only public HTTP or HTTPS URLs or signed S3 URLs are supported
  • Dynamic Ingest accepts up to 100 text tracks at a time
  • Dynamic Ingest only adds/appends text tracks to video’s vtt list even for a reencode request. customer should use the CMS API to manage them (update/delete)
  • mime_type not supported. but the captions should be UTF-8 encoded and have a mime-type of "text/vtt"
