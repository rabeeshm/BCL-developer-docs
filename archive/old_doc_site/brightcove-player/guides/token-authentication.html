In this topic, you will learn about using signed URLs and token auth to prevent end users from being able to play back video content without proper authorization. This topic applies to video-on-demand (VOD) content.

Overview

If you are delivering premium or confidential content, you will be looking for all ways to ensure it remains secure. You do not want end users to be able to copy video URLs or continue to play back content without the proper authorization. Brightcove supports VOD signing of RTMP, HLS, and DASH urls.

Note the following limitations:
  • HLS and DASH are supported for accounts on Akamai only.
  • Signed MP4 and Smooth URLs are not supported.
  • For accounts that have been enabled for Dynamic Delivery, URL signing for all VOD types is supported, currently on Akamai only.

Requirements

The following table lists the requirements for using Token Auth with HLS content in the Brightcove Player. Note that all requirements must be met to use Token Auth.
