In this topic you will learn how you can view player information from the Player Information Modal window.

View player information

You display the player information by right-clicking on the player and selecting Player Information, as shown in this screenshot.

Note: On mobile you can use the long press to open the modal.

Once you select Player Information you will see the modal window containing information about the player, as shown in this screenshot.

Note: Only Brightcove supplied plugins that are part of the the configuration will appear in the Plugins section. If you add a Brightcove plugin in an HTML page's JavaScript, it will NOT appear in the list. Customer developed plugins, even if listed as part of the player's configuration, will NOT appear.

Close the modal window

You close the modal either pressing the ESC key or by clicking on the close icon at the top-left.

Alter the context menu

You can alter the menu that appears on right-click, called the context menu. To do this you must alter the configuration of your player. The tool to perform the configuration change is curl. The two changes you can make, and how to affect them, are shown here.

Note: The curl statements below use the Player Management API to alter player configuration. In the curl statements you will see values of $EMAIL, $ACCOUNT_ID and $PLAYER_ID. You must either create environment variables by those names and assign corresponding values, or supply the literal values for the variables.

Prevent the menu appearing

If you wish to prevent the player information context menu from appearing at all, you can use the following curl statement.

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
      "debugger": false
  }' \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration
Note: Whenever you make a change to a player using curl, you need to re-publish the player. You can do that in Studio, and you will see your player is marked as needing to be published in the Status column:

You can also use curl to publish your player:

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request POST \
    https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/publish

Prevent the Brightcove menu option

If you wish to prevent the About Brightcove menu option from appearing, you can use the following curl statement.

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
       "debugger": {"brightcoveBranding": false}
  }' \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration
Note: Whenever you make a change to a player using curl, you need to re-publish the player. You can do that in Studio, and you will see your player is marked as needing to be published in the Status column:

You can also use curl to publish your player:

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request POST \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/publish
