Brightcove Player Sample: Related Videos from a Playlist

The page you are looking for doesn't live here anymore -- redirecting you to in 3 seconds.

The information contained in this document is not applicable for Perform customers.

In this topic, you will learn how to a Video Cloud playlist as the source of related videos. The related videos will appear when the viewer pauses the video or the video completes playing. A maximum of nine videos can be displayed using the dimensions of the player and thumbnails used here, but those settings can be changed to fit your needs.

Development sequence

Here is the recommended development sequence:

  1. Use the embed_in_page player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use the new Video Cloud Studio to add the plugin and CSS to your player
  5. Replace the embed_in_page player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or embed_in_page

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause application logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe player, you can still use the embed_in_page player with a plugin that contains your JavaScript and CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation

In this sample, you will test with the In-Page embed code and then move the CSS and JavaScript into a plugin. Since this example does not interact with any page elements, an iframe implementation with a plugin will be detailed and used.

API/Plugin resources used

The following sections detail resources used from various APIs/plugins.

Brightcove Player API

  • catalog.getPlaylist(): Method to make a catalog request for the playlist with the specified id and invokes a callback when the request completes.
  • catalog.load(): Method to load the video object into the player. Also populates the mediainfo property.
  • play(): Starts the video currently in the player.

Plugins

  • Display Overlay Plugin

The player

Start the video, then when you pause or when the video ends you can see thumbnails that represent the videos in the desired playlist. You can then select a new video to play.

How it works

The following sections explain the player, JavaScript and CSS code used in the example.

Create the player

To create a player and assign a video to it, follow these steps:

  1. Using Video Cloud Studio, in the Players module, create a new player or use an existing one.
  2. In the Media module, select a video and publish it with this player.
  3. Copy the In-Page embed code and paste it into a new HTML file.

Login to your Perform Studio account and create a new video player. Open the player's properties page and edit the Video Content section. Here you can specify a URL for the video you wish to publish. Videos can be in HLS and MP4 formats.

The player code

Your player code should look similar to the following:

<video id="myPlayerID"
    data-video-id="4511351172001"
    data-account="1752604059001"
    data-player="2fc88f47-3e0f-45bc-9a11-97d9d2f22392"
    data-embed="default"
class="video-js" controls></video>

The HTML code

No HTML code is added to the page.

The JavaScript code

Add the following JavaScript to your page code:

  • Lines 6-8, 10-16: Create variables for later use. Note that lines 57-59 create an HTML <div> element and assign it an ID. This element will be assigned the thumbnails and then used as content in the overlay. Line 54 assigns the ID of the desired playlist you wish to use for the related videos to a variable.
  • Lines 9, 27: Wait for the player to be ready for programmatic interaction.
  • Lines 18, 27: Reads a playlist using the player's catalog object and defines the callback function.
  • Line 22: Reduces the number of videos to a maximum of nine.
  • Line 24: Calls the createVideoList function which creates the thumbnails and assigns an HTML anchor tag to each.
  • Line 26: Appends the element of thumbnails/anchors to the dynamically created <div>.
  • Lines 30-36: Uses standard notation to call the Overlay Plugin and create an overlay for when the player pauses or ends. Even though the overlay will be displayed on pause and end, you will need to only listen for the pause event as this event is also dispatched when ended is dispatched.
  • Lines 45-48: This function is called when a viewer clicks on a thumbnail to play a new video. The function loads the correct video from the playlist then plays the video. It is VERY important that this is defined as a function expression, where a variable is assigned a function, rather than the more common function declaration notation. This is done so the function definition will be hoisted to global page scope when this code is used in a plugin.
  • Lines 56-66: Creates and HTML element. This function was called to create the <div> used as part of the overlay.
  • Lines 73-98: This function is passed the an array of videos and from the list builds the HTML for the overlay. The following screenshot shows the raw HTML dynamically created. Note that an anchor tag is created and the loadAndPlay function is called with the corresponding playlist item as an argument.
<script src="//players.brightcove.net/1752604059001/2fc88f47-3e0f-45bc-9a11-97d9d2f22392_default/index.min.js"></script>
  <!--include the Overlay plugin's JavaScript -->
  <script src="//players.brightcove.net/videojs-overlay/1/videojs-overlay.min.js"></script>

  <script type="text/javascript">
  var loadAndPlay,
    myPlayer,
    returnedPlaylist;
    videojs('myPlayerID').ready(function(){
      myPlayer = this;
      var playlistID = 4609649081001,
      htmlElement = '',
      // build a div that will be used as overlay content
      overlayDiv = createEl('div', {
        id: 'bclsOverlay'
      });
      // get the playlist using the catalog
      myPlayer.catalog.getPlaylist(playlistID, function(error, playlist){
        //deal with error
        returnedPlaylist = playlist;
        // be sure no more than 9 videos in playlist
        returnedPlaylist = returnedPlaylist.slice(0,9);
        // call function to use video objects to build div using thumbnails
        htmlElement = createVideoList(returnedPlaylist);
        // append div of thumbnails to overlay element
        overlayDiv.appendChild(htmlElement);
      });
      // use overlay plugin to display related videos on pause and video end
      // note that only starting on pause event as on ended event a pause event is also dispatched
      myPlayer.overlay({
        overlays: [{
          align: 'top',
          content: overlayDiv,
          start: 'pause',
          end: 'play'
        }]
      });
    });
    /**
     * loads and plays a video
     * this function called with thumbnails in
     * the overlay are clicked
     * @param {integer} idx the index of the video object in the videoData array
     */
    loadAndPlay = function(idx) {
      myPlayer.catalog.load(returnedPlaylist[idx]);
      myPlayer.play();
    }
    /**
     * create an element
     *
     * @param  {string} type - the element type
     * @param  {object} attributes - attributes to add to the element
     * @return {HTMLElement} the HTML element
     */
    function createEl(type, attributes) {
      var el,
        attr;
      el = document.createElement(type);
      if (attributes !== null) {
        for (attr in attributes) {
          el.setAttribute(attr, attributes[attr]);
        }
        return el;
      }
    }
    /**
     * create the html for the overlay
     *
     * @param {array} videoData array of video objects
     * @return {HTMLElement} videoList the div element containing the overlay
     */
    function createVideoList(videoData) {
      var i,
        iMax = videoData.length,
        videoList = createEl('div', {
          id: 'videoList'
        }),
        videoHeader = createEl('h1', {
          style: 'color:#666600;font-size: 2em;margin-bottom:.5em'
        }),
        videoWrapper = createEl('div'),
        thumbnailLink,
        thumbnailImage;
      videoList.appendChild(videoWrapper);
      for (i = 0; i < iMax; i++) {
        thumbnailLink = createEl('a', {
          href: 'javascript:loadAndPlay(' + i + ')'
        })
        thumbnailImage = createEl('img', {
          class: 'video-thumbnail',
          src: videoData[i].thumbnail
        });
        videoWrapper.appendChild(thumbnailLink);
        thumbnailLink.appendChild(thumbnailImage);
      }
      return videoList;
    }
  </script>

Use a plugin

It is a best practice to use a plugin so that you can easily add the JavaScript code's functionality to multiple players.

Create the JavaScript file

Move the plugin's JavaScript code into a file and place it in an Internet accessible URL.

  • Lines 1, 96: Use the standard syntax to begin and end a Brightcove Player plugin.
  • Line 1: The first argument of the function call is the name of the function defined in the plugin code, in this case relatedPlaylist.
  • Line 96: The trailing semicolon is required.
The variable created for loadAndPlay function needs to be removed when creating the plugin. When this is done the function will be global to the page, and be able to be seen when a thumbnail is clicked.
videojs.plugin('relatedPlaylist', function() {
  var myPlayer = this,
  returnedPlaylist,
  playlistID = 4609649081001,
  htmlElement = '',
  // build a div that will be used as overlay content
  overlayDiv = createEl('div', {
    id: 'bclsOverlay'
  });
  // get the playlist using the catalog
  myPlayer.catalog.getPlaylist(playlistID, function(error, playlist){
    //deal with error
    returnedPlaylist = playlist;
    // be sure no more than 9 videos in playlist
    returnedPlaylist = returnedPlaylist.slice(0,9);
    // call function to use video objects to build div using thumbnails
    htmlElement = createVideoList(returnedPlaylist);
    // append div of thumbnails to overlay element
    overlayDiv.appendChild(htmlElement);
  });
  // use overlay plugin to display related videos on pause and video end
  myPlayer.overlay({
    overlays: [{
      align: 'top',
      content: overlayDiv,
      start: 'pause',
      end: 'play'
    }]
  });

  /**
   * loads and plays a video
   * this function called with thumbnails in
   * the overlay are clicked;
   * must be a function expression or scope
   * would be limited to plugin's function scope
   * and this function used by the HTML page
   * @param {integer} idx the index of the video object in the videoData array
   */
  loadAndPlay = function(idx) {
    myPlayer.catalog.load(returnedPlaylist[idx]);
    myPlayer.play();
  }

  /**
   * create an element
   *
   * @param  {string} type - the element type
   * @param  {object} attributes - attributes to add to the element
   * @return {HTMLElement} the HTML element
   */
  function createEl(type, attributes) {
    var el,
      attr;
    el = document.createElement(type);
    if (attributes !== null) {
      for (attr in attributes) {
        el.setAttribute(attr, attributes[attr]);
      }
      return el;
    }
  }

  /**
   * create the html for the overlay
   *
   * @param {array} videoData array of video objects
   * @return {HTMLElement} videoList the div element containing the overlay
   */
  function createVideoList(videoData) {
    var i,
      iMax = videoData.length,
      videoList = createEl('div', {
        id: 'videoList'
      }),
      videoHeader = createEl('h1', {
        style: 'color:#666600;font-size: 2em;margin-bottom:.5em'
      }),
      videoWrapper = createEl('div'),
      thumbnailLink,
      thumbnailImage;
    videoList.appendChild(videoWrapper);
    for (i = 0; i < iMax; i++) {
      thumbnailLink = createEl('a', {
        href: 'javascript:loadAndPlay(' + i + ')'
      })
      thumbnailImage = createEl('img', {
        class: 'video-thumbnail',
        src: videoData[i].thumbnail
      });
      videoWrapper.appendChild(thumbnailLink);
      thumbnailLink.appendChild(thumbnailImage);
    }
    return videoList;
  }
});

Deploy plugin and CSS

This related videos via playlist plugin leverages the Overlay plugin. Therefore, you must load the JavaScript and CSS files for both the custom plugin as well as the Overlay plugin.

To deploy the relatedPlaylist Plugin using the Players module, follow these steps:

  1. In the new Video Cloud Studio, open the Players module and locate the player.
  2. Click the link for the player to open the player properties.
  3. Locate the Plugins section and click Edit.
  4. For the JavaScript URL, enter the URL which specifies the location of where you saved your plugin's JavaScript.
  5. Click the + button to add another JavaScript file.
  6. For the JavaScript URL, enter the URL which specifies the location of the Overlay plugin's Javascript: 
    //players.brightcove.net/videojs-overlay/1/videojs-overlay.min.js
  7. For the CSS URL, enter the URL which specifies the location of where you saved your plugin's CSS.
  8. Click the + button to add another CSS file.
  9. For the CSS URL, enter the URL which specifies the location of the Overlay plugin's CSS: 
    //players.brightcove.net/videojs-overlay/1/videojs-overlay.css
  10. For the Name, enter relatedPlaylist.
  11. No options will be passed to the plugin, so simply enter empty braces for the options.
    {}
  12. Click Save and then Publish the player.

Use iframe

It is a best practice to use the iframe player implementation. Since you can create this functionality as a plugin, replace the embed_in_page implementation with the iframe implementation.

<iframe src='//players.brightcove.net/1752604059001/f7dac958-0743-4a48-afe1-bc3b30165a81_default/index.html?videoId=4607357817001' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
