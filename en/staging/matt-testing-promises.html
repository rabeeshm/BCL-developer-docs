Using JavaScript Promises

In this topic, you will learn how to use JavaScript Promises, which are returned by some Brightcove Player methods.

Simple Promise

Although it is NOT the purpose of this document to teach you how to write a JavaScript Promise, to understand how to use them it is instructive to see how a simple Promise is coded.

In the following a Promise is created that returns a string that states whether a task was completed or not.The key point is that either the Promise.resolve() or Promise.reject() is called. In this case a simple string is passed, but this could be an object.

Note: Although NOT demonstrated in this simple example, the Promise object represents the eventual completion (or failure) of an asynchronous operation, and its resulting value. In this way a Promise is similar in concept to a callback function.

let promiseToFinishTask = new Promise(function(resolve, reject) {

  //Business logic to set taskComplete boolean
  let taskComplete = true;

  if (taskComplete) {
    resolve('Yea, I am done');
  } else {
    reject('Not quite done');
  }
});

Next you want to see how to deal with the returned Promise information, which is what you will do with some Brightcove Player API method calls. The basic handling of a Promise follows:


  promiseName.then(
    // Function call for promise resolved
  ).catch(
    // Function call for promise rejected
  )

Hanlding the simple example above would appear as shown here:


promiseToFinishTask.then(function(fromResolve) {
  console.log(fromResolve);
}).catch(function(fromReject) {
  console.log(fromReject);
});

And either of the strings Yea, I am done or Not quite done would appear based on the business logic that set the taskComplete variable's value.
