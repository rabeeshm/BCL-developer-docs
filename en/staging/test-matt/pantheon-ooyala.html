Setup to Edit Ooyala Help on Pantheon

The following steps set up the process to edit the DEV instance of the ooyala-help site on Pantheon.

Generate SSK keys

  1. Open Terminal
  2. Move into /Users/YOURNAME/.ssh
  3. Enter ssh-keygen
  4. For the key name enter pantheon_rsa
  5. Enter passphrase, and save it somewhere
  6. Open the file pantheon_rsa.pub that was created by the keygen
  7. Copy the key. It will look something like this (with a bunch more characters):
    ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQCkG...4Smf mboles@Matthews-MacBook-Pro.local
  8. Browse to https://dashboard.pantheon.io and login
  9. Click the down arrow next to your name in the top-right of the UI and select My Dashboard
  10. Click on Account
  11. Click on SSH Keys
  12. Paste your key into the textarea and click Add Key

Clone DEV environment to your machine

  1. In Pantheon UI, navigate Organizations > Brightcove > Ooyala Help (Ooyala Help is the site)
  2. Click on Clone with Git
  3. Copy the Git command
  4. Return to Terminal
  5. Move into the folder in which you wish to clone the site, an ooyala-help folder will be created for you
  6. Enter ssh-add ~/.ssh/pantheon_rsa
  7. Paste the Git command, press Enter, and let the clone process finish

Add the site to your GitHub client

  1. Open GitHub App
  2. Select File > Add Local Repository...
  3. Point to the ooyala-help repo just cloned
  4. Click Add Repository

Make ssh-add permanent

You need to complete the following steps or you will have to do the ssh-add after every login.

  1. If you do not already have one, create a file named config in the .ssh folder.
  2. In the file, add the following (you could have other config info already in there): 
    Host *
  UseKeychain yes
  AddKeysToAgent yes
  IdentityFile ~/.ssh/pantheon_rsa
  3. Reboot your machine.
  4. Open Terminal and enter ssh-add -K ~/.ssh/pantheon_rsa.
  5. From now on when you reboot, your connection to ooyala-help won't prompt for a password.

TWO ways to update the site

If you are making updates to the GitHub repo on your local machine you follow standard GitHub workflow to complete the changes.

If you are making changes to the Dev site via the Drupal UI, you must do the following to move the changes from Dev to Test:

  1. Manually backup both the Dev and Test instances. (You don't have to do this, but a good idea just in case.)
  2. Move to the Test instance, and click on Database/Files.
  3. Be sure the From this Environment dropdown has Development selected. (Don't think we even have any other options.)
  4. Check the Run update.php and Clear Caches checkboxes.
  5. Click the Clone the Database & the Files from Development into the Test Environment button.
  6. Updates now implemented in the Test environment.
