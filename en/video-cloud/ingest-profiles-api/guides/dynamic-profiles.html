Dynamic Profiles (beta)

In this topic, you will learn how to create and use dynamic ingest profiles.

This technology in currently in beta testing.

Introduction

Dynamic profiles is a technology developed by Brightcove to analyze the source video during transcoding and generate an optimum set of renditions and bitrates based on the complexity of the video and a set of bounding parameters provided (minimum number of renditions, max bitrate, etc.). A comparable industry term for this technology is "per title encoding".

Limitations

  • Server-Side Ad Integration (SSAI) will not work on renditions generated through Dynamic profiles

Enabling an account

You will need to enable your account for Dynamic Delivery by contacting Customer Support (If you haven't already). After that send an email to dynamic-profiles-beta@brightcove.com to enable your account to use Dynamic Profiles.

Create dynamic profiles

Once your account is enabled, you can add dynamic profiles using the Ingest Profiles section of the Studio Admin pages. The steps that follow will walk you through creating your first dynamic profile.

  1. Login to Studio.
  2. Go to the Admin>Ingest Profiles section.
  3. Click the New Profile button.
  4. Click on the Advanced Profiles tab:
  5. Replace the JSON code in the editor with the following, replacing the highlighted fields: 
    {
  "name": "Your Profile Name",
  "account_id": "Your Account Id",
  "digital_master": {
    "rendition": "passthrough",
    "distribute": false
  },
  "renditions": [],
  "packages": [],
  "dynamic_origin": {
    "renditions": [
      "default/audio64",
      "default/dynamic",
      "default/audio128",
      "default/audio96"
    ],
    "dynamic_profile_options": {
      "min_renditions": 3,
      "max_renditions": 6,
      "max_resolution": {
        "width": 1280,
        "height": 720
      },
      "max_bitrate": 2000
    },
    "images": [
      {
        "label": "thumbnail",
        "height": 90,
        "width": 160
      },
      {
        "label": "poster",
        "height": 720,
        "width": 1280
      }
    ]
  }
}
  6. Click Save, and you now have a dynamic profile. Try uploading a video in the Upload module, selecting your new profile to process it. This should provide a set of renditions that are optimized for playback on the most commonly used platforms such as desktop and mobile devices.

Playback testing

While we expect the playback experience to be seamless regardless of static or dynamic profiles, this technology is in beta, and we encourage you to test this and let us know (dynamic-profiles-beta@brightcove.com) of any instances where there is noticeable differences in quality.

  "digital_master": {
    "rendition": "passthrough",

After you create the profile, upload some content using it, and try it out on a large screen monitor or TV.

Dynamic Profiles and DRM

If your account is enabled for DRM, by default your optimally produced renditions will be stored and delivered encrypted.

Benefits

Below are some of the benefits of switching to dynamic profiles:

  • Improvement in quality of service – Because many videos will be encoded at a lower bitrate, viewer performance should improve: faster load times and less buffering.
  • Reduction in storage and bandwidth costs – Most videos will be encoded at a lower bitrate and an optimum set produced as needed based on the input. Based on our research, on average total size of the renditions are reduced by 40% and less bandwidth consumed by the end user much lower resulting in lower storage and bandwidth costs.
  • Potential increase in visual quality – Because we are analyzing each video and creating content-specific profiles, video with a higher complexity will be encoded with higher bitrates. As a result, there should be less distortion than with a basic static profile.
