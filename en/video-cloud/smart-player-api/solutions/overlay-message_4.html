Smart Player API Samples: Overlay Message

This is a more complicated version of Overlay Message 1. The overlay functionality is identical, but this version is fully functional on iPads — the basic version shows the overlay on iPads, but it does not respond to taps.

The Player

A custom player is used for this sample. It contains only a VideoDisplay without media controls. Here is the BEML code used to create the player:


<Runtime>
  <Layout id="application" width="480" height="270">
    <VideoDisplay id="videoPlayer" togglePlaybackOnClick="false" />
  </Layout>
</Runtime>

Note: custom players can be created only in Professional or Enterprise accounts.

How it is done

The functionality to implement the overlay here is exactly the same as in Overlay Message 1, so refer to the notes there on how it is done.

The limitation of the basic sample is that the overlay message is not tappable on iPads, because the native media controls in the iPad player swallow all tap events on the player screen. To overcome that, we created a custom player here (see the BEML code above) that has no media controls, just a videoDisplay. We then added custom controls of our own using HTML and CSS. The controls interact with the player through the API, using play() and pause() to start and pause the video, and the video properties and PROGRESS events to display time and progress in the timeline.

Note: we did not implement seeking by clicking on the playhead well here, though that is not hard to do — see the Thumbnails on Timeline sample for an example of how to do that.
