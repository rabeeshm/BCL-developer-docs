brightcove.api.modules.CaptionsModule(experience)
API class for handling captions. Captions are displayed in the player after being loaded or changed here. All captions must currently be in Timed Text Markup Language. These are found in a DFXP file, as described here.

Currently the CaptionsModule supports a subset of the DFXP standard. Supported attributes include xml:lang, begin, and end. 
var APIModules = brightcove.api.modules.APIModules;
var captionsModule = experience.getModule(APIModules.CAPTIONS);

getLanguages(videoID, callback)
Invokes the callback function, passing in the languages used in the captions for a video.
If no video ID is given, then the current video is selected.
If no captions are found, then an empty Array is returned.

loadDFXP(data, videoID)
Asynchronously load the DFXP file and associate with a video ID. If a video ID isn't given, the DFXP file is loaded for the current video. After the DFXP file is loaded and parsed, the captions will be displayed whenever the given video is played. Events are dispatched for load error or success.

DFXP files must be hosted on the same domain as the containing page or must be delivered with an appropriate Access-Control-Allow-Origin header.

Currently the CaptionsModule supports a subset of the DFXP standard. Supported attributes include xml:lang, begin, and end. 
var APIModules = brightcove.api.modules.APIModules;
var CaptionsEvent = brightcove.api.events.CaptionsEvent;
var captionsModule = experience.getModule(APIModules.CAPTIONS);
captionsModule.addEventListener(CaptionsEvent.DFXP_LOAD_SUCCESS, loadHandler);
captionsModule.addEventListener(CaptionsEvent.DFXP_LOAD_ERROR, errorHandler);
captionsModule.loadDFXP("http://domain/captions.dfxp");
Parameters:
{string} data
Either the URL for the DFXP file or a string representation of the DFXP file itself. In plugins the string representation is required.
{number} videoID
(optional) The video ID to associate the captions with. If a video ID is not provided, the DFXP file is loaded for the current video.

setLanguage(language)
Sets the language to use for captions. If a language hasn't been specified through this API, "en" is used. This language needs to have corresponding lang elements within the loaded DFXP files for any captions to be displayed. This language should be a ISO 639-1 Code as specified here.

Pass a null value to turn off captions. 
var APIModules = brightcove.api.modules.APIModules;
var captionsModule = experience.getModule(APIModules.CAPTIONS);
captionsModule.setLanguage("en");
Parameters:
{string} language
The current language for captions.
