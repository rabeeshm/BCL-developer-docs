Inherits from NSObject
Declared in BCOVIMASessionDeliveryPolicy.h

Overview

Policy object to determine when playback sessions should be delivered by the BCOVIMASessionProvider.

Subclass this class and override the signalWithValue: method to return a signal that sends a value when the specified playback session should be delivered.

immediateDeliveryPolicy

Returns a BCOVIMASessionDeliveryPolicy implementation that instructs the BCOVIMASessionProvider to deliver its sessions immediately, rather than wait for any sort of ad response. This means that if autoPlay is enabled, the content may begin playing before or concurrently with the retrieval of ad data.

waitForVASTResponseIfBeforeCuePointOfTypeExistsDeliveryPolicy:

Returns a BCOVIMASessionDeliveryPolicy implementation that instructs the BCOVIMASessionProvider to wait until a VAST response has been received before delivering each playback session, but only if the playback session’s video has a cue point whose position is kBCOVCuePointPositionTypeBefore and whose type matches the specified type. If no cue points with position kBCOVCuePointPositionTypeBefore and type matching the specified type exist, the session is delivered immediately.

+ (instancetype)waitForVASTResponseIfBeforeCuePointOfTypeExistsDeliveryPolicy:(NSString *)type

Parameters

type

Type property of the cuepoint.

Discussion

Returns a BCOVIMASessionDeliveryPolicy implementation that instructs the BCOVIMASessionProvider to wait until a VAST response has been received before delivering each playback session, but only if the playback session’s video has a cue point whose position is kBCOVCuePointPositionTypeBefore and whose type matches the specified type. If no cue points with position kBCOVCuePointPositionTypeBefore and type matching the specified type exist, the session is delivered immediately.

This will prevent the content from playing before ad data is available, so as to avoid a jarring “flash” of content before prerolls begin.
