In this topic, you will learn how to create a tabbed navigation for multiple playlists with a player.

Development sequence

Here is the recommended development sequence:

  1. Use the In-Page embed player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use Video CloudPerform Studio to add the plugin and CSS to your player
  5. Replace the In-Page embed player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or In-Page embed

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or In-Page embed implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the In-Page embed player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe code, you can still use the In-Page embed code with a plugin for your JavaScript and a separate file for your CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation

This sample uses the In-Page embed code. You can also move the CSS and JavaScript into a plugin. Because this solution interacts with the HTML page code, we will not use the iframe implementation.

How it works

Create the player

To create a player and assign several playlists to it, follow these steps:

  1. In the Players module, create a new player or use an existing one.
  2. Edit the Settings section, and enable the Display Playlist setting.
  3. If you want, you can add your player dimenstions. Notice that, by default, these dimensions are adjusted to include the playlist. In this example, we will use CSS to control the player and playlist sizes.
  4. Click Save and then Publish the player.
  5. Copy the In-Page embed code and paste it into a new HTML file.
For details on how to get this embed code, see the Quick Start to Brightcove Player document.

The JavaScript code

Add the following JavaScript code to load each playlist as the associated tab is selected in the navigation:

For Perform customers, you will create a playlist using the player's playlist() method along with JSON to define the videos in the playlist. For details, see the Using Playlists document.
  • Lines 2-8: Define an array of playlist Ids, along with other variables.
  • Lines 10-13: Use the player's ready() method to indicate when the player is loaded on the page. Get a reference to the player. Call the processTab() method, passing an index value of 0 for the first tab and playlist in the navigation.
  • Lines 15-19: In the processTab() function, reset the tabs so that none of them are selected/highlighted.
  • Line 17: Highlight the selected tab in the navigation.
  • Lines 21-26: Load the selected tab's playlist into the player.
  • Lines 28-34: In the resetTabs() function, turn highlighting off for all of the tabs.
<script id="pageScript" type="text/javascript">
  var myPlayer,
      playlistIds = ["4450721964001","2805100167001","2764931905001","1754200320001","1754200353001"],
      playlistIdsLength = playlistIds.length,
      tabs = document.getElementsByClassName("button"),
      currentTab,
      currentTabName,
      playlistNames = [];

  videojs("video_1").ready(function(){
      myPlayer = this;
      processTab(0);
  });

  function processTab (index) {
      resetTabs();
      document.getElementById("tab" + index).setAttribute("style","background:#08c;color: #00FFFF; border-bottom: 1px solid #00FFFF;");
      loadPlaylist(playlistIds[index]);
  };

  function loadPlaylist (currentId) {
      myPlayer.catalog.getPlaylist(currentId, function(error, playlist){
          myPlayer.catalog.load(playlist);
          myPlayer.playlist.currentItem (0);
      });
  };

  function resetTabs () {
      var i,
          iMax = tabs.length;
      for (i = 0; i < iMax; i++) {
          tabs[i].setAttribute("style", "background: #0000cc;color: #fff;")
      }
  }
</script>

The stylesheet

The following styles control the container for the tabbed navigation along with the player and playlist:

  • Lines 2-7: set the styles for the entire page.
  • Lines 10-17: set the styles for the container which wraps the tabbed menu, player and playlist.
  • Lines 20-50: define the tabbed navigation.
  • Lines 53-78: make the player responsive.
  • Lines 81-124: set the layout and styles for the tabbed playlist when the media query detects a small screen device, like a phone.
  • Lines 127-161: set the layout and styles for the tabbed playlist when the media query detects a medium screen device, like a tablet.
  • Lines 164-187: set the layout and styles for the tabbed playlist when the media query detects a large screen, like a desktop browser.
Lines 72, 73 and 99 use the CSS !important declaration to ensure these values take precedence over later rules. This is needed to make the player responsive in IE11.
Be careful with the pseudo selectors. An extra space in line 63 will cause the player to not be visible. For example, you do not want this .myplayer: after {
