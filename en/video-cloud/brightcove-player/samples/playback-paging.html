In this topic, you will learn how to implement playlist paging when retrieving data from the Playback API.

The information contained in this document is not applicable for Perform customers.

Overview

To page through a set of videos in a playlist, you can use the following request URL parameters:

  • limit - defines the number of videos to return from the Playback API
  • offset - sets the number of videos to skip in a playlist from the Playback API

Development sequence

Here is the recommended development sequence:

  1. Use the embed_in_page player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use Video CloudPerform Studio to add the plugin and CSS to your player
  5. Replace the embed_in_page player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or embed_in_page

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe code, you can still use the embed_in_page code with a plugin for your JavaScript and a separate file for your CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation

In this example, you will use the Advanced (In_Page) embed code. You will not create a plugin for the player, since the code interacts with the Next Videos button on the page.

API resources used

The following sections detail resources used from various APIs.

Brightcove player API methods

  • catalog.getVideo(): Method to make a catalog request for the video with the specified id and invokes a callback when the request completes.
  • catalog.load(): Method to load a video into the player.
  • playlist(): Method to assign an array of video objects to a player's playlist. Note that this method can also be used to retrieve the playlist currently playing in the player, but will not be used in this way in this document.

Playback API

The Playback API will be used to return videos from the specified playlist.

If you are new to using the Playback API, review the Overview for the Playback API document.

In order to make a request to the Playback API, you will need a Policy Key. There are three ways to obtain a policy key:

  1. Every Brightcove player is automatically assigned one. You can find instructions for getting this in the Policy Keys document. Since policy keys are good account-wide, you can use it regardless of whether that player is embedded on the page.
  2. You can generate a basic policy key using the Policy Keys Quick Start
  3. You can make a request to the Policy API

The player

This example uses a Brightcove player to display a video in the player along with the first 3 videos in the playlist. Select the Next Videos button to display the next 3 videos in the playlist. When you reach the last set of videos in the playlist, then the videos displayed wrap around to the beginning of the playlist.



    Playback API request:

    Notice that the request to the Playback API uses the limit and offset parameters to implement paging.

    The JavaScript code

    Here, you will add code to control the functionality of the Next Videos button.

    1. Add the following JavaScript to your page code:
      • Lines 2-18: Define variables. Use your own values for the Policy Key, Account Id and Playlist Id.
      • Line 21: Gets a reference to the player when it is ready.
      • Lines 23-27: Make a call to the Playback API to get the total number of videos in the playlist.
      • Line 29: Makes a call to the Playback API to get the first set of videos in the playlist and loads them into the player.
      • Lines 30-36: Add a listener to the Next Videos button. When selected, make a call to the Playback API to get the next set of videos in the playlist and load them into the player.
      • Lines 42-46: Make a request to the Playback API to get a set of videos from the playlist.
      • Line 49: Loads the first video object into the player.
      • Line 50: Loads the current set of video objects into the playlist.
      • Lines 61-67: Set the Playback API request URL. First time, all videos are returned for the playlist. After that, the next set of videos are returned from the playlist, using the limit and offset parameters.

      • Lines 75-111: Make the Playback API request using the request URL along with the Policy Key in the header. 

      <script type="text/javascript">
  var myPlayer,
      requestData,
      mediaData,
      apiRequest,
      apiBaseURL = 'https://edge.api.brightcove.com/playback/v1/accounts/',
      policyKey = your policy key,
      accountId = your account Id,
      playlistId = your playlist Id,
      videoData = [],
      mediaType,
      offsetValue = 0,
      limitValue = 3,
      firstTime = true,
      requestURL,
      eNextButton = document.getElementById("nextButton"),
      eApiRequest = document.getElementById('apiRequest'),
      playlistLength;

  videojs("myPlayerID").ready(function(){
      myPlayer = this;
      // set up data for Playback API request
      requestData = setRequestData();
      // make the Playback API request to get playlist data
      getMediaData('playlist', requestData, function (mediaData) {
          // console.log("mediaData= ",mediaData);
          playlistLength = mediaData.videos.length;
          // console.log("playlistLength= ",playlistLength);
          displayPlaylist();
          eNextButton.addEventListener("click",function(){
            offsetValue += 3;
            if (offsetValue >= playlistLength) {
              offsetValue = 0;
            }
            displayPlaylist();
          });
      });
  });

  function displayPlaylist() {
    // set up data for Playback API request
    requestData = setRequestData();
    // console.log("requestData= ",requestData);
    mediaType = 'playlist';
    // make the Playback API request to get playlist data
    getMediaData(mediaType, requestData, function (mediaData) {
        // console.log("mediaData= ",mediaData);
        // add the newest videos list to the player as a playlist
        myPlayer.catalog.load(mediaData.videos[0]);
        myPlayer.playlist(mediaData.videos);
    });
  }

  /**
   * sets up the data for the API request
   */
  function setRequestData() {
    console.log("setRequestData");
    var endPoint = '';

    if (firstTime) {
      firstTime = false;
      requestURL = apiBaseURL +  accountId + '/playlists/' + playlistId;
    } else {
      requestURL = apiBaseURL +  accountId + '/playlists/' + playlistId + '?offset=' + offsetValue + '&limit=' + limitValue;
      eApiRequest.innerHTML = requestURL.trim();
    }

    return requestURL;
   }

   /**
    * request data from the Playback API
    */
    getMediaData = function(mediaType, requestURL, callback) {
    var httpRequest = new XMLHttpRequest(),
      responseData,
      parsedData,
      // response handler
      getResponse = function() {
        try {
          if (httpRequest.readyState === 4) {
            if (httpRequest.status >= 200 && httpRequest.status < 300) {
              responseData = httpRequest.responseText;
              parsedData = JSON.parse(responseData);
              console.log("parsedData",parsedData);
              switch (mediaType) {
                case 'video':
                  // do nothing
                  break;
                case 'playlist':
                  callback(parsedData);
                  break;
              }
            } else {
              alert('There was a problem with the request. Request returned ' + httpRequest.status);
            }
          }
        } catch (e) {
          alert('Caught Exception: ' + e);
        }
      };
    // set response handler
    httpRequest.onreadystatechange = getResponse;
    // open the request
    httpRequest.open('GET', requestURL);
    // set headers
    httpRequest.setRequestHeader('Accept', 'application/json;pk=' + policyKey);
    // open and send request
    httpRequest.send();
  };
</script>
