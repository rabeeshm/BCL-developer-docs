Player Debugger Plugin

The debugger plugin enables your player to display a debug dialog overlay on top of the video element. This is a default plugin and is automatically loaded with the Brightcove player. However, you can choose not to load it. For details on how to not load a default plugin when you create a player, see the Player Plugin Overview document.

This document walks you through the basics of using the player debugger.

Load trigger

The debugger code is always present in the player. To load it follow the appropriate trigger:

  • On desktops press F2 (on a Mac keyboard that is option-Fn-F2)
  • On mobile devices use a three finger tap

After the trigger you will see the debugger open in the top-right of the browser window.

Once the debugger is opened you will see the messages that have been, and are, written to the log.

Purpose

The debugger is not so much a tool for development, but more for a technical person working with a user troubleshooting a player issue. The debugger can be a help to the developer, as it can augment the development tools in your favorite browser.

UI

The debugger has nine buttons across the top for manipulation of the debugger itself and the displayed data.

The first five buttons are used to toggle display of the corresponding message types in the debugger. Those are:

  • Debug
  • Info
  • Warning
  • Error
  • Profile

Later in this document you will see methods you can use to display each type of message.

The remaining four buttons are:

  • Email Current Log - This opens your email client with the current log as the body of mail.
  • Toggle Size: This toggles the debugger between the smaller and larger formats.
  • Clear - Clears the log.
  • Close - Closes the debugger.

Methods

The methods documented in this section can be used in your JavaScript, or entered in the console.

The following methods are always available to write a message of the given type to the debugger log:

  • videojs.log.debug(message) - Adds a debug message
  • videojs.log.warn(message) - Adds a warn message
  • videojs.log.info(message) - Adds an info message; this one is equivalent to videojs.log() itself
  • videojs.log.error(message) - Adds an error message

The following methods are available once the debugger has been loaded:

  • videojs.log.resize() - Toggles the size of the debugger window between large and small
  • videojs.log.clear() - Clears the current output in the debugger window
  • videojs.log.move() - Moves the debugger window the four corners of the browser
  • videojs.log.profile(message) - Adds a profile message
