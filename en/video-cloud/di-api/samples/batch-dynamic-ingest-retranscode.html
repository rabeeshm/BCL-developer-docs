Dynamic Ingest - Replace Multiple Videos

This is a sample of eplacing of videos using the Dynamic Ingest API. The Dynamic Ingest API call is exactly the same as it would be for ingesting new videos - the only difference is that you do not need to make a prior call to the CMS API to create the video object in the Video Cloud system and get an id for it. If the source video file at the specified URL is the same one originally ingested, you will simply get a new set of renditions. If the source file is new, you will be replacing the existing video. All videos will remain playable with existing renditions until retranscoding is complete.

To get access tokens and make the API calls, it uses a proxy built by Brightcove Learning services.

Getting Credentials

To get a client_id and client_secret, you will need to go to the OAuth UI and register this app:

  • https://studio.brightcove.com/products/videocloud/admin/oauthsettings

You can also get your credentials via CURL or Postman - see:

  • Get Client Credentials Using CURL
  • Get Client Credentials Using Postman

Rate limiting

No more than 100 concurrent Dynamic Ingest jobs are allowed. The best way to avoid violating this rule is to create an app that receives notifications from the Dynamic Ingest system as well as submitting ingest requests, so that you can keep track of how many jobs are running and how many have completed - see Fallback / Retry Strategy. This sample implements rudimentary rate limiting by pausing for 30 minutes after submitting 100 requests.
