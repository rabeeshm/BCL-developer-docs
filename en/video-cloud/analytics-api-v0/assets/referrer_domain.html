Overview

The referrer_domain dimension is the top private domain of the referrer URL on a page where events are collected for activity metrics (this URL is sent to Collector with the source parameter). The referrer_domain dimension only includes metrics from player instrumentation events.


referrer_domain in the API

Referrer URL

API Value

http://blog.brightcove.com/en/ 

(any URL with a valid public domain)

brightcove.com 

(top private domain for the URL)

http://192.168.0.0 

(any private domain or IP address)

non_public_domain

(direct traffic, invalid URL)

null


Metrics

The following metrics are available for the referrer_domain dimension:

  • player_load
  • video_impression
  • video_view
  • video_percent_viewed
  • video_seconds_viewed


Metadata

There is no metadata collected for a referrer_domain.


Rollup Data

Report Data

Example requests/responses for filtering referrer_domain dimension reports by video or player are fairly redundant, and so are not included here. Note (as always) that particular summary metrics are only returned where they are configured (e.g. filtering a referrer_domain dimension report by a specific video does not return player_load metrics). See Dimensions and Metrics for the current supported dimension + metric combinations


Functional Expectations

Given that [player_load, video_impression, video_view, video_engagement] events have been collected for referrer http://blog.brightcove.com/en

    And I make a call for report data with dimensions=referrer_domain

    Then I get a report that includes "referrer_domain": "brightcove.com"


    And I make a call for rollup data with where=referrer_domain==brightcove.com

    Then I get a rollup report that includes trending data for [player_load, video_impression, video_view, video_percent_viewed, video_seconds_viewed] metrics


Given that [player_load, video_impression, video_view, video_engagement] events have been collected for multiple referrers

    And I make a call for report data with dimensions=referrer_domain

    Then I get a report with referrer_domain items with [player_load, video_impression, video_view, video_percent_viewed, video_seconds_viewed] summary data


Given that [player_load, video_impression, video_view, video_engagement] events have been collected for referrers with player 567

    And I make a call for report data with dimensions=referrer_domain&where=player==567

    Then I get a report of referrer_domain items with [player_load, video_impression, video_view, video_percent_viewed, video_seconds_viewed] summary data for player 567


Givent that [video_impression, video_view, video_engagement] events have been collected for referrers with video 890

    And I make a call for report data with dimensions=referrer_domain&where=video==890

    Then I get a report of referrer_domain items with [video_impression, video_view, video_percent_viewed, video_seconds_viewed summary] data for video 890
