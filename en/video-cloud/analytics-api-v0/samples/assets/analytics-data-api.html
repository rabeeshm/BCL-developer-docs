Introduction

The Analytics Data API is a means to access data collected from various sources (e.g. the Video Cloud player instrumentation). Except where specified, this document assumes a valid auth token. In addition, nearly all examples here use the videocloud domain, but this document is valid for the appcloud domain was well.

Service URL

The endpoint for the Analytics Data API is:

https://data.brightcove.com/analytics-api/<domain>

where domain is one of videocloud or appcloud. (Domains namespace data -- TODO: link to more info)


Authorization

The Analytics Data API requires a valid token to return data. Auth tokens should be included in the HTTP request header:

Authorization:Bearer <auth_token>

If abcdef is the auth token for Video Cloud account 12345, a cURL request might look like:

curl --header "Authorization:Bearer abcdef" "https://data.brightcove.com/analytics-api/videocloud/accounts/12345"

About LA Auth Tokens

Auth tokens are provided to publishers as they are added to the LA program. These auth tokens work for a single account, and so the account and auth token provided must match. The Analytics Data API responds with 401 Unauthorized if the auth token is invalid for the request or if an auth token is not included.


API Basics

(This section assumes the presence of a valid auth token for account 1234.)

The account id must be included in all Analytics Data API requests:

https://data.brightcove.com/analytics-api/videocloud/accounts/1234


Getting Data

By nature, this data is time-based, and covers a defined set of dimensions.

The Analytics Data API returns data in two distinct ways:

  1. Rollup data for a dimension (or the account).
  2. Report data for a dimension (or the account), broken down dimensionally.


Response Status Information

HTTP Status

Explanation

You've got data! Everything is awesome!

401 Not Authorized

Make sure you've got a valid auth token for your account, and that you're including it in your request correctly.

Make sure your request is valid, including your account id.

Our system is currently TU. Please wait and retry your request later.

Rollup Data

ex. https://data.brightcove.com/analytics-api/videocloud/accounts/1234

These requests return "rollup" data for a specific dimension, without breaking it into constituent parts. This information is detailed, and contains metrics broken down over time (e.g. a time series record of video_view events from the past day, broken down by hour, or a record of video_engagment events from the past week, broken down by percentage of a video's timeline), as well as any metadata we may have collected about that dimension. (These are called "rollups" because they collect constituent information into a detailed whole; contrast with report data, in which data is broken down dimensionally but does not include as much detailed information about the specific dimension itself.)


Structure

Responses depend on the domain, but the basic document structure is approximately:

{

    "account": `1234`,

    "metrics": {

        ...

     }   

}

where metrics contains a set of domain-specific metrics data objects. For example, a videocloud rollup might contain

{

    "account": "1234",

    "metrics": {

        "video_view": {

            "series": {

                "step": 3600000,

                "type": "time",

                "values": [ ... ],

                "start": 1357077600000

            },

            "alltime": 5467

        },

        ...

    }

}

which is the video_view metric for account 1234. The series.values array contains trending video_view data for the time range; see the video_cloud domain documentation for more information. TODO


Parameters

Rollups don't have any special parameters in addition to the general set. A request for rollup data for account 1234 in the default date range would look like this:

     https://data.brightcove.com/analytics-api/videocloud/accounts/1234

The where parameter can be used to request rollup data a specfic dimesion; a request for rollup data for video 8967 (in account 1234) using the default date range would look like this:

    https://data.brightcove.com/analytics-api/videocloud/accounts/1234?where=video==8967

    (encoded: https://data.brightcove.com/analytics-api/videocloud/accounts/1234?where=video%3D%3D8967)


Report Data

ex. https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report

Report requests return a list of data for a given dimension type that contribute to a data set. To put it a little more concretely, a simple report with the video dimension returns the set of videos that contributed to an account's analytics data in a given date range.


Structure

Responses depend on the domain, but the basic document structure is approximately:

{ 

    "account": "1234",

    "item_count": 1542

    "items": [

        { 

            "<dimension>": "<dimension_id>",

            ...

        },

        {   "<dimension>": "<dimension_id>",

            ... 

         },

        ...      // etc.

    ]

}

where item_count is the total number of items in the data set, and items is the list of dimensional data in the response. Each item contains a summary of data for the dimension in the specified date range. (To get trending or other data over time, get rollup data for your account or the specific dimension you are interested in.)

This example:

https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report?dimensions=video

Returns a list of all videos that contributed to the analytics data collected for account 1234 in the default range. Similarly, we could look at the list of all players for account 1234:

https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report?dimensions=player

For further segmentation, the where parameter can be used to subset the data:

https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report?dimensions=video&where=player==8675

this report will return the list of videos that contributed to the analytics data collected for the player 8675 in account 1234.

Using where in this way allows us to answer questions like

Of all the players where I published the "I Knew You Were Trouble" video (id 8888) which player had the most views in my account (1234) the last 30 days?

with a request like

https://data.brightcove.com/analytics-api/videocloud/accounts/1234/report?dimensions=player&where=video==8888&sort=video_view
