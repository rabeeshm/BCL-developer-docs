Overview

The device_type dimension is the type of device detected from the user-agent captured when events were collected for activity metrics.



device_type in the API

The device types currently tracked include:

  • mobile - included mobile phones
  • tablet - includes iPads, Android tables, etc.
  • tv - includes Smart TVs, set-top boxes, and game consoles.
  • desktop - includes personal computers.
  • other - for user-agents which are not mapped to any of the above.

Device detection is done using DeviceAtlas device detection software.



Metrics

The following metrics are available for the device_type dimension:

  • player_load
  • video_impression
  • video_view
  • video_percent_viewed
  • video_seconds_viewed



Metadata

There is no metadata collected for device_type dimensions.

Example requests/responses for filtering device_type dimension reports by video or player are fairly redundant, and so are not included here. Note (as always) that particular summary metrics are only returned where they are configured (e.g. filtering a device_type dimension report by a specific video does not return player_load metrics). See Dimensions and Metrics for the current supported dimension + metric combinations.



Functional Expectations

Given that [player_load, video_impression, video_view, video_engagement] events have been collected for user-agent _Mozilla/5.0 (iPhone; CPU iPhone OS 6_0_1 like Mac OS X) AppleWebKit/536.26 (KHTML, like Gecko) Version/6.0 Mobile/10A523 Safari/8536.25_

    And I make a call for report data with dimensions=device_type

    Then I get a report that includes "device_type": "mobile"


    And I make a call for rollup data with where=device_type==mobile

    Then I get a rollup report that includes trending data for [player_load, video_impression, video_view, video_percent_viewed, video_seconds_viewed] data for the device_type "mobile"
