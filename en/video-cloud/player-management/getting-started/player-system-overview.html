Brightcove Player Management Overview

Brightcove Player and the accompanying Player Management API provide a platform on which to build a flexible, extensible video playback experience.

Brightcove Player

The key strengths of Brightcove Player are:

  • Lightweight player loads in HTML5 mode by default, with a fall back to Flash if HTML5 is not supported
  • Rich plugin environment to add features and functionality
  • Use of HTML, JavaScript and CSS to customize player appearance
  • Two player implementations for different environments
  • Rich set of features such as cue points, captions, localization, advertising, etc.
  • Native player SDKs
  • Plays both Video Cloud library videos and videos with Internet accessible URLs

Further information on the Brightcove Player can be found in the Brightcove Player documentation.

Player Management API

The player management API's features fall into two major categories. Those categories and detailed strengths are listed here:

  • Centralized management
    • Automatic version updates
    • Create, update and activate players via API
    • Update thousands of players at once
    • Player versioning and rollback
  • Optimized performance
    • Pre-compilation of player assets
    • Embed thumbnail or video URLs in player
    • Deliver plugins through the same CDN

Further information on the Player Management API can be found in the Brightcove Player Management documentation.

Video Cloud Studio

Players can be created, customized and managed, with some limitations, using Video Cloud Studio. Video Cloud Studio provides a easy to use interface to perform these tasks.

The Getting Started with the New Video Cloud Studio document provides an excellent starting point to learn about Video Cloud Studio.

Player Management API

The Brightcove player and the associated APIs provide you with both a lightweight and highly customizable player and a system for deploying players in multiple ways. Managing players via the Player Management API

The Player Management API allows you to create player configurations, which are the specifications for players. You can not only create multiple players, but also can also use the configuration of one player in multiple ways by using child players of a parent player. You can assign different media, styles and plugins to each child player, while still maintaining the parent/child relationship to the original player, so changes made to that player are inherited by all child players.

Player configurations are created in preview mode, which gives you additional flexibility in choosing when to publish them. Once you are satisfied with changes, you publish the player via a separate API call. Publishing creates a separate branch of the player configuration while retaining the preview branch. This means that you make and test modifications to the preview player and publish it to production only when you are satisfied with the changes.
