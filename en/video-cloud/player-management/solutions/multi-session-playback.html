Brightcove Player Sample: Multiple Session Playback

This topic describes how to use the new Brightcove player to update a session cookie during playback to maintain the current position.

When the player initially loads, a cookie will be checked for the last watched position of the video. If the cookie is present, the video will begin playing at that point. When the video finishes playing, the cookie will be set to zero.

iframe or embed_in_page

When developing enhancements for the new player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with the iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS.
  • The player is automatically responsive.
  • The iframe eases player use in social media apps (or whenever the player will need to "travel" into other apps).

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the page containing the player needs to communicate with the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Select implementation for this sample

In this case there is no need to interact with the containing page, so the best implementation is to create a plugin for use with an iframe player.

Even if your end result is going to be a plugin, it is recommended you first develop the code with an embed_in_page player, then convert the code to a plugin. That best practice is followed in this document.

How it works

The player code

The new Brightcove player code uses an HTML5 <video> tag to embed a video.

When you publish your player, you can use the embed_in_page code to put the player directly in an HTML page. For details on how to use this code, refer to the embed_in_page Guide.

Follow these steps to build the player code:

  1. Copy the embed_in_page URL and paste it into a browser to view the code.
  2. Create a new HTML document.
  3. From the embed_in_page code, move the <link> tag into the <head> section of your new document.
  4. Set the width and height properties for the player. By default, the player has 0 height.

The <head> section of your HTML document should appear as follows:

  1. The rest of the embed_in_page code goes into the <body> section of your HTML document.

  1. Optionally, you can add two HTML elements below the player to inject the initial value of the session cookie and the current value of the cookie as the video plays.

The JavaScript code

Before the closing </body> tag in your HTML document, create a <script> block where you will put the following custom code:

  1. Create a function that reads and parses the session cookie based on a key identifier passed in.
  2. Get a reference to the new Brightcove player. In this case, the <video> tag for your player has an id property of myPlayerID.
  3. Define your variables and read the session cookie with a key value of BC_position.
  4. If the session cookie exists, use the value stored in the cookie. Otherwise, use a value of 0 to start play from the beginning of the video.
  5. Next, inject the value of the cookie into a text area below the player.

  1. When all the data for the player has been loaded, set the current position using the currentTime() method and begin video playback using the play() method.

  1. Next, add a listener for the timeupdate event. This event is fired when the current playback position has changed. During playback, this will usually be every 15-250 milliseconds.
  2. In this function, check when the playback position has changed by a second and write this value to the session cookie.
  3. Inject the current value of the cookie into a text area below the player.

  1. Finally, add a listener for the ended event. When the video has ended, reset the cookie value to 0 to start the video from the beginning.

Using a plugin

This solution lends itself to using a plugin so that you can easily add this functionality to multiple players.

To create a plugin, follow these steps:

  1. Wrap your JavaScript code with the videojs.plugin() method. The first argument for this method is the plugin name. You will use this name when you add the plugin to your player.
  2. Since the plugin is a part of your player, you will get a reference to the player by using the this keyword.
  3. Be sure your plugin is placed on an Internet accessible URL.

The following curl statement can be used to associate the multi-session playback plugin with a player. As described in the Quick Start to Player Management, the environment variables ACCOUNT_ID, EMAIL and PLAYER_ID must be defined .

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
    "scripts": [
      "http://solutions.brightcove.com/bcls/video-js/new-player/plugins-doc-samples/multi-session-playback.js"
    ],
    "plugins": [{
      "name": "multiSessionPlayback"
    }]
  }' \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/player-management/$PLAYER_ID/configuration

If you have an existing plugin already associated with a particular player and you perform another PATCH operation, it will replace the scripts and plugins data. If you wish to add, and not replace, the curl statement will need to reflect cumulative data, that is, multiple scripts and plugin name objects. Alternatively, for properties that are datatyped as arrays, which both scripts and plugins are, you can use inheritance behavior to be overwrite, prepend or append additional values. For details on this functionality, see the Array fields section of the Embed APIs Guide.
