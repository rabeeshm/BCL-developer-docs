Managing Activity Lifecycle

As users interact with your Android app, the OS invokes callback methods in an Activity instance that correspond with different stages of its lifecycle.

If you cannot, or choose not to use the BrightcovePlayer, you will need to manage an Activity's lifecycle yourself.

Note: You can skip this topic if you are using the BrightcovePlayer, since you will not need to manage activity lifecycle manually.

If you are unfamiliar with Android's Activity lifecycle, get started with the Managing the Activity Lifecycle reference.

Methods and states

During the life of an activity, the Android OS calls a set of lifecycle methods, based on the users' interactions.

Lifecycle methods change the activity state as shown in the following table:

Method State Visible
onCreate() Created No
onStart() Started Yes
onResume() Resumed Yes
onPause() Paused Partially
onStop() Stopped Hidden
onRestart() Started Yes
onDestroy() Destroyed No

As an Activity transitions between states, there are only three states where an activity can exist for an extended period of time.

  • Resumed
  • Paused
  • Stopped

The other states (Created and Started) are transitional states, as the system quickly moves them to the next state.
