Quick Start to FTP Batch Provisioning: Home

The objective of this Quick Start is to give you a short, simple guide to follow to help get you started with using FTP batch provisioning. This Quick Start will give you a set of hands on steps you can work through on your own to upload, update, delete, re-transcode videos, and well as create manual and smart playlists.

After completing this Quick Start you should be able to:

  • Create FTP accounts with the proper permissions
  • Use your FTP account to upload video assets
  • Create an XML manifest file
  • Ingest videos to Video Cloud Studio
  • Update properties for your video assets
  • Create manual and smart playlists
  • Delete assets and playlists
  • Re-encode your videos
  • Define remote video and thumbnail assets

Audience

Anyone who wants to use FTP batch provisioning to process multiple assets and playlists.

Prerequisites

  • A Pro or Enterprise Video Cloud account
  • Some experience using Video Cloud

What is it?

  • FTP batch provisioning enables you to upload, update or delete multiple videos and playlists at once using an XML manifest file and the Video Cloud FTP server
  • You can use FTP batch provisioning to generate videos and playlists, which will appear in your Media Library in the Media module once the upload process is complete

Why use it?

  • Most actions that you want to perform on your assets and playlists via the Video Cloud Studio Media module can be done using FTP batch provisioning
  • You can upload large numbers of assets or make multiple edits quickly without accessing the Media module
  • Publishers use FTP batch provisioning rather than the Media module when they need to upload, modify settings for, or delete large numbers of videos and/or playlists
  • Perform initial ingestion of their existing video library into the Video Cloud platform

Steps to implement

  1. Create accounts and request proper permissions
  2. Gather and prepare your assets
  3. Create the XML manifest file
  4. Upload assets to the Video Cloud server using an FTP client
  5. Upload the XML manifest file to the Brightcove server using an FTP client
  6. Wait for processing to complete
