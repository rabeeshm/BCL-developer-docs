Using FTP Batch Provisioning to Change a Local Video to a Remote Video

In this topic you will learn how to use FTP Batch Provisioning to change a local video asset to a remote video asset, keeping the original system generated unique identifier and reference ID.

We assume that you will start with a local video asset.

Note: FTP Batch Provisioning is available only in Video Cloud Pro and Enterprise editions.

Get started

Let's first look at the basic code for an XML manifest file. You will add code to this file to ingest a remote video asset and update a local video asset. This file contains:

  • The Publisher ID associated with a Video Cloud account for a dedicated FTP user
  • An email address for your dedicated FTP user
<?xml version="1.0" encoding="utf-8" ?>
 <publisher-upload-manifest publisher-id="ENTER YOUR PUBLISHER ID HERE" preparer="FTP" report-success="TRUE">
   <notify email="ENTER YOUR FTP EMAIL ADDRESS HERE" />
 </publisher-upload-manifest>

This code is explained in the Quick Start to FTP Batch Provisioning document.

Change local to remote

Changing a local video asset to a remote video asset using FTP Batch Provisioning consists of the following steps:

  1. Identify your local video's system generated unique identifier and reference id
  2. Create an XML manifest file to define a remote video
  3. Update the XML manifest file to change your local video to a remote video
  4. Submit your XML manifest file
  5. Verify the system generated unique identifier and reference id for your remote video

Identify local

Identifying local video asset ids

In Video Cloud Studio, navigate to the Media module and select the All Videos link. To the right of the video name, locate the system generated unique identifier and the reference id.

With your video selected, use the Edit button to open the Edit Video dialog. In the Video Files section, notice that there are multiple renditions for this local video.

Create manifest file

Creating an XML manifest file to define a remote video

In the XML manifest file, add a <remote-asset> tag to define a remote video asset. For this tag, you will need the following:

  1. The URL of your hosted remote video
  2. Some information about your remote video, including size, duration and video-codec
<?xml version="1.0" encoding="utf-8" ?>
 <publisher-upload-manifest publisher-id="ENTER YOUR PUBLISHER ID HERE" preparer="FTP" report-success="TRUE">
   <notify email="ENTER YOUR FTP EMAIL ADDRESS HERE" />
   <remote-asset refid="birdRemote_asset_refid"
      display-name="Bird remote"
      type="VIDEO_FULL"
      remote-url="http://solutions.brightcove.com/bcls/ftpbpQuickStart/Bird_Titmouse_Remote.mp4"
      size="1678000"
      video-duration="8000"
      video-codec="H264"/>
</publisher-upload-manifest>

For detailed information about creating a remote video asset, see the Creating Videos with Remote Video Files Using FTP Batch Provisioning and the Quick Start: Using FTP Batch Provisioning to Define Remote Assets documents.

Update manifest file

Updating the XML manifest file to change your local video to a remote video

Next, you will add code to the XML manifest file to update your existing local video to point to your remote video. Your local video will change to a remote video, while keeping the same system generated unique identifier and reference id.

In the XML manifest file, add a <title> tag which points to your local video's reference id and your remote video's asset id. Your file will appear as follows, with your respective values:

<?xml version="1.0" encoding="utf-8" ?>
 <publisher-upload-manifest publisher-id="ENTER YOUR PUBLISHER ID HERE" preparer="FTP" report-success="TRUE">
   <notify email="ENTER YOUR FTP EMAIL ADDRESS HERE" />
 <remote-asset refid="birdRemote_asset_refid"
      display-name="Bird remote"
      type="VIDEO_FULL"
      remote-url="http://solutions.brightcove.com/bcls/ftpbpQuickStart/Bird_Titmouse_Remote.mp4"
      size="1678000"
      video-duration="8000"
      video-codec="H264"/>
      
    <title name="Bird remote" 
 active="TRUE" 
         refid="birdLocal_title_refid" 
         video-full-refid="birdRemote_asset_refid" > 
         <short-description>Bird Titmouse Remote</short-description> 
    </title>
 </publisher-upload-manifest>

Submit manifest file

Submitting your XML manifest file to the Video Cloud server

Upload the XML manifest file to the Brightcove server to create a remote video and change your local video to this remote video.

  • Upload the XML manifest file to the Brightcove server using an FTP client.
  • Wait for the processing to complete. The XML manifest file should disappear from your FTP client.

When the ingestion is complete, you should see your remote video in the Media module in Video Cloud Studio.

Verify video asset

Verifying your remote video asset

In Video Cloud Studio, you should see your remote video associated with the same system generated unique identifier and reference id that were used for your local video.

In the Video Files section, notice that there are no renditions, since this video is now a remote asset.
