Deprecated Elements in FTP Batch Provisioning Manifest

This document contains deprecated elements that one might use in the FTP Batch Provisioning XML Manifest file. While these elements are functional, they are deprecated. For this reason, we suggest that all users reading this document avoid using elements listed in the table.

See the most current specification in FTP Batch Provisioning: Reference for the XML Manifest.

Title tag attributes

Name
 Description
flv-full-refid This title attribute contains the reference ID attribute of a full-length video file to include in a video. This reference ID could be either one previously entered into the system or one specified in an asset tag within the same manifest file. Since this attribute is deprecated; use video-full-refid instead.
color TRUE or FALSE. Indicates whether or not this video is in color. If you omit this attribute, its value will be considered to be TRUE. Not editable in the Media module. This attribute existed in Brightcove 2 and was eliminated in Brightcove 3. Instead, use custom metadata fields. If you do so, do not use the same metadata name, as it will conflict with the deprecated (but still existing) fields.
language The language the video is in. If you omit this attribute, no value will be assigned to this field. Not editable in the Media module. This attribute existed in Brightcove 2 and was eliminated in Brightcove 3. Instead, use custom metadata fields. If you do so, do not use the same metadata name, as it will conflict with the deprecated (but still existing) fields.
rating One of the following values:
TV-Y, TV-Y7, TV-Y7FV, TV-G, TV-PG, TV-14, TV-MA, MV-G, MV-PG, MV-PG13, MV-R, MV-NC17, Not-Rated, Not Rated, Parental-Advisory, Parental Advisory, No-Parental-Advisory, No Parental Advisory.Not editable in the Media module. This attribute existed in Brightcove 2 and was eliminated in Brightcove 3. Instead, use custom metadata fields. If you do so, do not use the same metadata name, as it will conflict with the deprecated (but still existing) fields.
release-date The date the video was released. Use the MM/DD/YYYY format. Not editable in the Media module. This attribute existed in Brightcove 2 and was eliminated in Brightcove 3. Instead, use custom metadata fields. If you do so, do not use the same metadata name, as it will conflict with the deprecated (but still existing) fields.
