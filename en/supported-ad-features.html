Supported Ad Features

This topic provides a quick reference to supported advertising features for various Brightcove products.

Overview

The table below provides a high-level overview of supported features. The sections that follow provide details.

Table 1: Overview
Smart Player Brightcove Player Mobile SDKs Once
Video Cloud Video Cloud / Perform Android (Video Cloud / Perform) iOS (Video Cloud / Perform) Once
VAST 2.0 Yes[1-1] Yes[1-6] Yes[1-6] Yes[1-6] Yes[1-2]
VAST 3.0 Yes[1-3] Yes[1-4] Yes[1-4] Yes[1-4] Yes
VPAID 1.0 Yes[1-5] No No No Yes[1-7]
VPAID 2.0 Yes[1-8] Yes[1-9] No No Yes[1-7]
DFP Ad Rules, Pods Yes[1-10] Yes[1-4] Yes[1-4] Yes[1-4] Yes
Freewheel Smart XML Yes Yes[1-11] Yes[1-11] Yes[1-11] Yes

Notes

  • [1-1] Flash only, via VAST Server Ad Source in Video Cloud
  • [1-2] Non-extended
  • [1-3] HTML only, via VAST Server Ad Source in Video Cloud; Flash and HTML, via DFP IMA3 Ad Source in Video Cloud
  • [1-4] Via Google IMA3 Plugin (Video Cloud / Perform)
  • [1-5] Flash only, via VAST Server Ad Source in Video Cloud
  • [1-6] To the extent that VAST 3.0 is backwards-compatible with VAST 2.0
  • [1-7] VPAID (.swf) is only supported on a pass-through basis. Once does not interact with or extract from the VPAID data at this time
  • [1-8] Flash only, via the DFP IMA3 Ad Source in Video Cloud
  • [1-9] Flash only, via the Google IMA3 Plugin (Video Cloud / Perform)
  • [1-10] Via DFP IMA3 Ad Source in Video Cloud
  • [1-11] Via Freewheel plugin (Video Cloud / Perform)
