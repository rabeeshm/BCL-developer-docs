Quick Start to Player Management

In this quick start you will perform the following tasks:

  • Obtain credentials to use the Player Management API
  • Create a player
  • Update a player by replacing the video it plays
  • Customize a player by adding a plugin
  • Display the configuration for a player

For details on the API used in this document, you can view the Player Management API Overview document and the Brightcove Player Management API reference.

OAuth proxy

When using the Player Management API with Video Cloud videos, security is implemented using OAuth2. To permit you to get started using the Player Management API as quickly as possible, the API calls in this quick start will use a security proxy running at http://solutions.brightcove.com:8003. If you browse to that URL, you will be redirected to an informational page on the proxy. Be aware that the proxy will only accept requests originating from localhost and *.brightcove.com domains.

Create player

Of course part of the Player Management API is to create a Brightcove player. This player will play a Video Cloud video. The fundamental call information for creating a video is:

  • HTTP method: POST
  • Endpoint: https://players.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/players

In the call body you supply a simple JSON object specifying the name of the player to be created, as well as the Video Cloud video to be placed in the player. To see the fields for which you can provide values during the POST, browse to the Player Configuration Guide.

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

  1. You will now use the proxy in conjunction with the following form to create a player. Be sure to alter the body so the name property reflects your choice for the player name and Video Cloud video the player should contain.

Update player

You can change a player you have created. In this section you will implement a very simple update, replacing in the player one Video Cloud video with another. You will see more complex update use cases later in this document. The fundamental call information for updating a video is:

  • HTTP method: PATCH
  • Endpoint: https://players.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/players/VIDEO_ID/configuration

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

Publish player

When you update a player you have a preview version of the player. You will want to publish the player before deploying for use. The published version differs from the preview version in these ways:

  • The JavaScript and CSS are minified, concatenated and inlined into the player directly
  • A low resolution version of the poster image is generated and inlined into the page to improve perceived load times on networks with a high request-setup time (i.e. cellular data networks)
  • The previous version of the player is saved so it can be recovered if problems are discovered after the update goes live

Note: On player creation (step 3 above) the player is also automatically published. That is why on player creation the JSON response included both published and preview information.

The fundamental call information for publishing a video is:

  • HTTP method: POST
  • Endpoint: https://players.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/players/VIDEO_ID/publish

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

  1. You will now use the proxy in conjunction with the following form to publish a player. Be sure the body contains no JSON.
  2. At the end of the current request URL you MUST append /publish to correctly publish the player.

Display configuration

    To debug and confirm work you have done it is often helpful to view a player's configuration. In this section you will display the configuration in these two different ways:

    • Alter the player's URL or preview URL
    • Use an HTTP GET method
  1. Copy either the player's URL or preview URL (at this time, the preview URL will display the more interesting configuration which reflects the plugin being used) and at the end of the URL change the index.html to config.json. You will see the returned JSON displayed in the browser.
  2. You can also use an API call to display the player configuration. The fundamental call information for retrieving player configuration is:

    • HTTP method: GET
    • Endpoint: https://players.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/players/VIDEO_ID/configuration

Delivery system APIs

The Delivery System APIs allow for the management and deployment of a group of files, called a repository. These files are managed through REST APIs and through git. These APIs shouldn't be needed by most people when creating or editing players, but they can be a very interesting set of APIs to use for other purposes. If you'd like to try them out, you can do so here. For a hands-on introduction, try out the Quick Start to the Delivery System.
