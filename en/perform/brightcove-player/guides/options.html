Configure the Player

You can configure the Brightcove player using the Player Management API or directly in your web page as discussed in this topic.

IP Restrictions Outside of North America

When utilizing IP restrictions, there is a very important player configuration that needs to be done for players used outside of North America. IP restrictions can prevent a Video Cloud video from reaching the Brightcove Player. A player configuration can fix this issue. The configuration change must be done using the aforementioned Player Management API. There is a section in the Player Configuration Guide that details how to perform this configuration change.

iframe options

With the iframe implementation, there are a few query parameters that you can add to the Brightcove player URL.

autoplay

If autoplay is true, the video will start playing as soon as page is loaded (without any interaction from the user). NOT SUPPORTED BY APPLE iOS DEVICES. Apple blocks the autoplay functionality in an effort to protect it's customers from unwillingly using a lot of their (often expensive) monthly data plans. A user touch/click is required to start the video in this case.

src="//players.brightcove.net/1752604059001/default_default/index.html?videoId=4029697544001&autoplay"

src="//players.brightcove.net/1752604059001/default_default/index.html?autoplay"

Any of the following values will set autoplay to true:

autoplay
autoplay=true
autoplay=1
autoplay=yes

Setting autoplay to any value will turn it on for your player - even setting autoplay=false. To turn off autoplay, simply remove it from the player URL.

Note: You can also turn autoplay on for a player by using the Players module in Studio or through the Player Management API.

language

The language query parameter sets the language of the player. The player must have the desired language added to the player for this to function correctly.

src="//players.brightcove.net/1507807800001/EkRkYsAjx_default/index.html?videoId=4607746980001&language=de"

See the Localize Brightcove Player document for complete details.

preload

The preload query parameter can be used to set whether preloading should be used:

src="//players.brightcove.net/1752604059001/default_default/index.html?videoId=4029697544001&preload=yes"
src="//players.brightcove.net/1752604059001/default_default/index.html?preload=yes"

You can also use the preload() method.

videoId

You can dynamically change the video that is played by using the videoId query parameter as follows:

src="//players.brightcove.net/1752604059001/default_default/index.html?videoId=4029697544001"

For details about setting the video Id or using a reference Id, see the Assign Video Programmatically document.

In-Page embed options

The Video.js embed code is simply an HTML5 video tag, so for many of the options you can use the standard tag attributes to set the options.

<video controls autoplay preload="auto" ...>

Alternatively, you can pass in an object with the player options as the second argument in the javascript setup function.

videojs("example_video_1", { "controls": true, "autoplay": false, "preload": "auto" });

 

Note on video tag attributes:

With HTML5 video tag attributes that can only be true or false (boolean), you simply include the attribute (no equals sign) to turn it on, or exclude it to turn it off. For example, to turn controls on:

WRONG

<video controls="true" ...>

RIGHT

<video controls ...>

The biggest issue people run into is trying to set these values to false using false as the value (e.g. controls="false") which actually does the opposite and sets the value to true because the attribute is still included. If you need the attribute to include an equals sign for XHTML validation, you can set the attribute's value to the same as its name (e.g. controls="controls").

 

Following is a list of options you can use with the In-Page embed code:

controls

The controls option sets whether or not the player has controls that the user can interact with. Without controls the only way to start the video playing is with the autoplay attribute or through the API.

<video controls ...>
or
{ "controls": true }

autoplay

If autoplay is true, the video will start playing as soon as page is loaded (without any interaction from the user). NOT SUPPORTED BY APPLE iOS DEVICES. Apple blocks the autoplay functionality in an effort to protect it's customers from unwillingly using a lot of their (often expensive) monthly data plans. A user touch/click is required to start the video in this case.

<video autoplay ...>
or
{ "autoplay": true }

preload

The preload attribute informs the browser whether or not the video data should begin downloading as soon as the video tag is loaded. The options are auto, metadata, and none. For HLS playback technology the settings mean slightly different things and noted below.

  • auto (default): Start loading the video immediately (if the browser agrees). Some mobile devices like iPhones and iPads will not preload the video in order to protect their users' bandwidth. This is why the value is called auto and not something more final like true.

    For the HLS playback technology, auto means the player downloads the m3u8 manifest file and only 1 segment instead of 3-4, as was the case before version 5.3 of Brightcove Player.

  • metadata: Load only the meta data of the video, which includes information like the duration and dimensions of the video.

    For the HLS playback technology, metadata is the same as auto.

  • none: Don't preload any of the video data. This will wait until the user clicks play to begin downloading.

    For the HLS playback technology, none means player will no longer load anything, including the m3u8 manifest, until the user presses play.

<video preload ...>
or
{ "preload": "auto" }

poster

The poster attribute sets the image that displays before the video begins playing. This is often a frame of the video or a custom title screen. As soon as the user clicks play the image will go away.

<video poster="myPoster.jpg" ...>
or
{ "poster": "myPoster.jpg" }

To disable the poster image (video still) for the Brightcove Player, simply set the poster attribute to one of the following values:

poster=""
or
poster="null"
or
poster="false"

loop

The loop attribute causes the video to start over as soon as it ends. This could be used for a visual affect like clouds in the background.

<video loop ...>
or
{ "loop": "true" }

width

The width attribute sets the display width of the video.

<video width="640" ...>
or
{ "width": 640 }

height

The height attribute sets the display height of the video.

<video height="480" ...>
or
{ "height": 480 }
