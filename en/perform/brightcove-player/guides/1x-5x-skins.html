Skins and Player Versions

The Brightcove Player skins are quite different between versions 1.x and 5.x (released in early 2016). The following summarizes the situation:

  • Brightcove Player instances created after the release of 5.x will, of course, use the new skin, named luna.
  • Players that existed before the global update to version 5.x were left on the 1.x skin, named graphite.

An easy way to determine if your player has the old (graphite) or new (luna) skin is to check the initial play button, often referred to by its CSS name, vjs-big-play-button.

  • The first screenshot below shows the 1.x skin with a rectangular big play button located near the top left of the player.
  • The second screenshot shows the 5.x skin with the a circular big play button centered in the player. Also note that the title of the video is automatically displayed in the 5.x skin.

A centered, circular button is not a guarantee of the 5.x skin as it is possible to use CSS to customize the 1.x button, but it is a likely indicator.

Use 5.x player with 1.x skin

If you wish to use a newly created 5.x with a 1.x skin, you must use the Player Management API to do this. The following curl command enables you to use the v5.0.2 player without the new skin. You must substitute appropriate values for YOUR_EMAIL, YOUR_ACCOUNT_ID and YOUR_PLAYER_ID.

curl \
 --header "Content-Type: application/json" \
 --user YOUR_EMAIL \
 --request PATCH \
 --data '{
    "player": {
     "template": {
       "version": "5.0.2"
     }
    },
    "skin": "graphite",
    "compatibility": true
 }' \
 https://players.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/players/YOUR_PLAYER_ID/configuration

In most cases, you will want to publish the player after a PATCH, which can be done with the following curl statement. Again, you will need to substitute correct values where indicated.

curl \
 --header "Content-Type: application/json" \
 --user YOUR_EMAIL  \
 --request POST \
 https://players.api.brightcove.com/v1/accounts/YOUR_ACCOUNT/players/YOUR_PLAYER_ID/publish

Update existing player to 5.x skin

As stated above, when existing 1.x players were upgraded to 5.x the skin was left as a 1.x skin. It is possible that you want that player to use the 5.x skin. To do this, you will need to use the Player Management API to "upgrade" your skin.

The following mini-app (as a CodePen) presents a form to complete and perform this upgrade. DO NOT USE THIS FOR 1.x PLAYERS. For details on how this app works, see the Brightcove Player Sample: Player Skin Updater document.

See the Pen Brightcove Player Skin Updater by Brightcove Learning Services (@bcls) on CodePen.
