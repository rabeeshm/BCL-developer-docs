embed_in_page Guide

This topic describes how to use the embed_in_page code.

Introduction

The embed_in_page code permits you to have the player exist in the the HTML page directly, not in an iframe. This offers the benefits of ease of accessing the player and associated properties and events. By using the non-iframe player you lose the ease of use of the player as a single, contained unit in social media applications and sharing.

Attributes

Numerous attributes are available for the <video> tag to provide additional information about how a player should behave. The table below details available attributes.

Tag attributes that can only be true or false (marked boolean), you simply include the attribute (no equals sign) to turn it on, or exclude it to turn it off.

RIGHT: controls

WRONG: controls="true"

Attribute Description
data-account The account ID, sometimes also called the Publisher ID
data-embed Displays information if you are using embeds (parent-child player relationships)
data-player The player ID
data-playlist-id The ID or reference ID of the playlist in the player
data-video-id The ID or reference ID of the video in the player
autoplay boolean - Video will start playing as soon as page is loaded (without any interaction from the user). NOT SUPPORTED BY APPLE iOS DEVICES.
class Standard HTML attribute that will be assigned the video-js value by default
controls boolean - Shows player controls
height Sets the display height of the video
loop boolean - Causes the video to start over as soon as it ends.
muted boolean - Mutes the video's sound
poster Sets the image that displays before the video begins playing. This is often a frame of the video or a custom title screen. As soon as the user clicks play the image will go away.
preload boolean - Informs the browser whether or not the video data should begin downloading as soon as the video tag is loaded.
width Sets the display width of the video

Delay instantiation

There are a few use cases where you will want to use the embed_in_page code, but delay player instantiation. To do this, you have to alter the <videos> tag's attributes and use the bc() method to instantiate the player.

Use cases where this technique is helpful are, for example, lightbox styles and single page web apps where the page with the player is not initially instantiated.

Modify video tag

To do this you CANNOT use all the <videos> tag's attributes, or the player will be instantiated. You must remove three of the attributes from the normal In-Page Embed code. The attributes that need to be removed are:

  • data-account
  • data-player
  • data-video-id

This leaves the following attributes in the <video> tag:

  • data-embed
  • class
  • controls (optional)

The reason the attributes must be removed as any of those three attributes remain it will cause Brightcove Player to be instantiated.
