Brightcove Player Sample: Navigate on Video End

This topic shows you how to use a Brightcove player that displays an overlay on video end, which when clicked navigates the page to a new URL.

Plugins used

Note: This example uses open source video.js plugins:

  • videojs-overlay : For details, refer to the Display Overlay Plugin document.

Development sequence

Here is the recommended development sequence:

  1. Use the In-Page embed player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use Video CloudPerform Studio to add the plugin and CSS to your player
  5. Replace the In-Page embed player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or In-Page embed

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or In-Page embed implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the In-Page embed player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe code, you can still use the In-Page embed code with a plugin for your JavaScript and a separate file for your CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation

In this example, you will use Studio to create a video player that implements the Overlay plugin.

An iframe implementation will not work in this case since the new URL page would be contained within the dimensions of the iframe player.

The player

This example displays an overlay when the video finishes playing. When the user selects the link contained in the overlay, the page navigates to the new URL.

How it works

The following sections explain the player, JavaScript and CSS code used in the example.

Create the player

To create a player and assign a video to it, follow these steps:

  1. In the Players module, create a new player or use an existing one.
  2. In the Media module, select a video and publish it with this player.Assign video content to the player and publish it.
  3. Copy the In-Page embed code and paste it into a new HTML file.

For details on how to get this embed code, see the Quick Start to Brightcove Player document.

Adding a plugin

It is a best practice to use plugins so that you can easily add the overlay functionality to multiple players. Using Studio, you can specify JavaScript and CSS files to implement the Overlay plugin.

  1. Copy the following URL for the Overlay plugin's JavaScript file.
    2. //players.brightcove.net/videojs-overlay/1/videojs-overlay.min.js
  2. In the Players module, scroll down to the Plugins section and paste the JavaScript URL for the Overlay plugin.

  3. Copy the following URL for the Overlay plugin's CSS file.
    4. //players.brightcove.net/videojs-overlay/1/videojs-overlay.css
  4. In the Plugins section, paste the CSS URL for the Overlay plugin.

  5. Assign a name to your plugin and specify the plugin options in JSON format. The options below define the following:
    • The text and URL displayed in the overlay
    • To display the overlay when the video ends
    • To hide the overlay when the video starts playback

  6. In the Plugins section, enter your plugin name along with the plugin options.

    7. Note: For details concerning overlay options see the Display Overlay Plugin document.

Use iframe

It is a best practice to use the iframe player implementation if possible.

In this example, the iframe implementation will not work, since we want the entire page to navigate to the new URL. With the iframe implementaion, the contents of the new URL page would be contained within the dimensions of the iframe player.
