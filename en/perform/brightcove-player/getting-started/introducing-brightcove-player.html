The New Brightcove Player

Video Cloud users can use the new Brightcove Player with existing media assets. The Brightcove Player has great new features you can deploy today, either alongside or to replace your existing Smart Players. The new Brightcove Player is based on new technology, so customizations or plugins you may have relied on with the Smart Player will work differently and have to be reimplemented. Some features have been replaced or changed in important ways. Assembled here are materials and links that will help you understand the benefits of using the new Brightcove Player and what you should consider as you try it out.

The Brightcove Player System

There are multiple ways of working with the new Brightcove Player:

Players Module
The Players Module is part of the next generation Video Cloud Studio and will replace the old Publishing Module. You can use the Players Module to do everything you need to:
  • Create players
  • Style players
  • Add plugins
  • Get publishing code
The Brighcove Player
The documentation for The Brighcove Player explain the player itself in detail, including how to create plugins, how to use built-in plugins, and a full reference for the player API.
The Player Management API
The Player Management API offers you a programmatic way to create, modify, and maintain players as an alternative to using the Players Module. These documents will be of interest mainly to developers who want to create integration between Video Cloud and a CMS.
