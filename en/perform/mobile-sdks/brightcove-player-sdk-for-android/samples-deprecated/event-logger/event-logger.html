Using the Event Logger

This topic describes how to use the event logger functionality to observe events on Brightcove components.

This document is based on a working example with code snippets throughout.

  • You can view the complete MainActivity.java file.

Create event loggers

One of the strengths of the Brightcove Player SDK for Android is the sophisticated event model. Being able to observe event information on Brightcove components makes application development more understandable and debugging easier. You can use the EventLogger class to observe event information.

Observe events

Once the event loggers are created you will want to leverage them to observe event information. You can view the system log in Android Studio by selecting the Devices | logcat option in the DDMS (Dalvik Debug Monitor Server) window. When you run your application from within Android Studio, you can look through the log for the tags you used when creating your event logger objects. This can be a daunting task, as there may be many, many lines to look through. A better option is to create a filter as described next.

To make viewing of the events reported by the event logger objects you created easier, it is beneficial to create a filter.

  • In the logcat tab, expand the filters drop-down menu and select the Edit Filter Configuration option.

  • In the Create New Logcat Filter dialog, add the value for the by Log Tag option. This must match the tag entered (the third, String argument) when creating the objects.

When you run your application, you will see event information for the corresponding object.
