Understanding Brightcove Video Views

This topic explains how to get started with the Brightcove Player SDK for Android when building a native app to play video content.

Start with an activity

When you create an app using the Brightcove Player SDK for Android, you should start with either the BrightcovePlayer or the BrightcovePlayerFragment class.

The advantages of using one of these classes include:

  • They provide basic lifecycle management and full-screen support.

    This means, when your app is paused, the class will keep track of where playback stopped. When your app resumes, it will start up again where it left off.

  • They setup an event logger and write all events to logcat.
  • They dispatch events to communicate with the Brightcove plugins.
  • They wrap a BaseVideoView class, which is discussed in the next section.

Choose a video view

The BaseVideoView class provides the basic functionality for the video view, and its subclasses do the following:

  • They are designed to be dropped into your XML Layout file to replace Android's VideoView widget.
  • They provide public methods to mirror those from Android's VideoView widget.
  • They provide additional logic for playlists.

The subclasses for the BaseVideoView can be divided into the following categories:

Basic video playback

  • BrightcoveVideoView

    This uses the Android MediaPlayer, similar to Android's VideoView. This is the view you will want to use for basic video playback.

HLS video playback

  • SeamlessVideoView

    Also known as the Brightcove HLS Player, the SeamlessVideoView provides greatly improved support for HLS and seamlessly stitching videos together. This is where it gets it's name.

    The SeamlessVideoView internally uses a VisualOn player, which requires additional native libraries to be included in your app and has additional costs. Please contact your Brightcove account manager for pricing details.

  • BrightcoveExoPlayerVideoView

    This uses Google's Exo Player. This is another good choice for playing HLS videos.

Video playback with animation

  • BrightcoveTextureVideoView

    This uses the TextureView for rendering. This view is used if you want to support animation, like having the video play while it is opening up from a thumbnail.

For more details about these classes, refer to the Brightcove Player SDK for Android Reference, found in the top navigation.

Define player controls

The BrightcoveMediaController class provides player controls similar to the Brightcove Web Player to provide a consistent user experience across platforms. You can use the out-of-the-box Brightcove player controls, or customize them with your own colors, styles and buttons.

The Brightcove player controls provide support for captions, video rewind and full-screen mode. You can also select the animation style for showing and hiding the controls.
