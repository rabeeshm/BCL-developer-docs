Once Error Notifications

The Once system can notify you when there is an issue with one of your media items. This document details the meaning of the messages you receive.

Configuring notifications

Details of how you want the notifications delivered can be specified in your account configuration - contact your Brightcove representative. You can specify the URL you want notifications sent to, and the method (POST, GET, or PUT).

Notification format

Notifications are sent as JSON. Here is a sample notification with an explanation of the fields:

{
  "___version" : "1.0",
  "errorCode" : "500",
  "errorMessage" : "There was an error in the Media System",
  "jobId" : "{GUID}",
  "passThruMetadata" : "clob",
  "status" : "Error"
}
Field explanation
Property Description
jobid Customer identifier for the transcode job.
passthrumetadata Any metadata set for passthrough sent on the initial ingest request
status Completed or Error
errorCode Error code if the process did not succeed
errorMessage Brief data around why a particular asset failed

Error code reference

The following table represents the services and errors that our back end system will notify on.  However, on a per-domain basis we can customize what errors you get notifications for.

We also support the ability to define custom messages and error codes per domain.  This will allow you to use specific codes in the notification to help facilitate automation and internal notification on specific errors.

Keep in mind that if a single custom message is added it will override all the default configurations and a custom message configuration will have to be added for each of the supported service error types.
