Production Default Notifications

The following matrix  represents the services and errors that our back end system will notify on.  However on a per domain basis we can customize what errors they get notifications for.

We also support the ability to define custom messages and error codes per domain.  This will allow the customer to use specific codes in the notification to help facilitate automation and internal notification on specific errors.

Keep in mind that if a single custom message is added it will override all the default configurations and a custom message configuration will have to be added for each of the supported service error types.
