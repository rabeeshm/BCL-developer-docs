Dynamic Ad Parameters (DAPs)

Available Dynamic Ad Parameters (DAP's) Ad Request Content Beacon Ad Beacon

mediaitem.title

  • String value of mediaitem.title

mediaitem.tremor.title

  • String value of mediaitem.tremor.title (necessary when using Tremor as the ad network) Media Item Title with space URL encoded using %20.

mediaitem.keywords

  • String value of mediaitem.keywords

mediaitem.categories

  • Comma separated list of the string values of categories

mediaitem.language

  • The string language value for mediaitem

mediaitem.rating

  • The string rating value of mediaitem

mediaitem.guid

  • String value of mediaitemguid

mediaitem.foreignkey

  • String value of mediaitemforeignkey

mediaitem.duration

  • Content file duration in seconds based on the total length of content

catalogguid

  • Catalogguid where mediaitem is located

catalogname

  • String value of catalog name. Following characters CAN NOT be used in catalog names: < > : " / \ | ? or *

catalog.categories

  • Comma separated list of the string values of categories

applicationguid

  • Player application GUID

applicationname

  • Player application name

mediaitem.adposition

  • Integer value for the ad position, e.g., 1, 2, 3 for pre, mid, post

mediaitem.adpositiontime

  • Time in seconds where ad is being played

randomnumber32

  • A unit random number

timestamputc

  • A 32 bit representation of the servers current time in unix time (UTC)

useragent

  • String of the useragent of the client

ipaddress

  • User’s IP address

referringHost

  • String of the referring host

referringURL

  • URL Encoded representation of HTTP Header Referrer

countrycode

  • String of the ISO country code for this user

postalcode

  • String of the user’s postal code (if available)

xm."key"

  • extended metadata. The "key" can be whatever values you used when ingesting extendedmetadata for an asset.

visitguid

  • UM visit GUID. for use as stream view uniqueness id.

connectionspeed

  • geoService bandwidth value. (mobile, broadband, etc)

carrier

  • geoService carrier/mobile-carrier value

isp

  • geoService isp-name value. (carrier)

dma

  • geo DMA value

msa

  • geo MSA value (Not available with Net Acuity)

region

  • geo region value

state

  • geo state value.

latitude

  • To accommodate ad providers that require lat/long to be sent with ad tag request (Videology / TWC & more)

longitude

  • To accommodate ad providers that require lat/long to be sent with ad tag request (Videology / TWC & more)

beacon.quartile.text

  • The quartile text, e.g. Start, FirstQuartile, Midpoint, ThirdQuartile, Complete.

beacon.quartile.value

  • The raw value of the quartile in seconds.

beacon.interval.value

  • The interval in seconds when the beacon is being fired.

xfp.correlator

  • (DFP) Random # correlator specific to DFP. used to implement competitive exclusions, including in cookieless environments. (unique per page view)

xfp.pod

  • (DFP) Represents a pod within a video. Pass &pod=1 for first pod, &pod=2 for second pod, and so on.

xfp.ppos

  • (DFP) Position within a pod. Pass &ppos=1 for the first position, &ppos=2 for the second position, and so on.

xfp.vpos

  • (DFP) Whether ad request is being sent from pre-roll,mid-roll or post-roll.

xfp.scor

  • (DFP) Integer generated for each video stream that reflects unique streams (so each piece of content); the number needs to be the same within a stream and unique within a pageview.

xfp.ipaddress

  • CBS has asked that we carry the IP value into eachstandard and optimizedpod call.

xfp.ss_req

  • CBS has asked for this to be added to all pod/optimized pod calls made within their domains. It is required for all initial and subsequent ad tags for the UM-to-XFP session.
  • (Viacom - SmartXML) Ad creative i.d. implemented for Viacom's use case in beaconing (see OV-136).
  • (Viacom - SmartXML) Ad pod i.d. implemented for Viacom's use case in beaconing (see OV-136).
  • (Viacom - SmartXML) Position in the pod. No path. Programmatically generated based on the position sequence of the adCreativeId within a temporal pod (aka slot). (see OV-136).
