Legacy Ingest Options

These are legacy ingest options for Once. These are not options for new customers.

FTP Content Ingest

FTP Content Ingest offers an alternative asset upload process by allowing you to quickly transfer assets via FTP using nearly any client. This service layer permits you to upload a media file and metadata file to our cloud where it is ingested, transcoded and made available for publishing.

Process

During setup, we build and internally maintain a 'Drop Directory' associated with the Brightcove Media domain, content catalog(s) and your user account. You upload your media file as well as a mapped XML metadata file of the same file name and an .xml extension. When both the media and XML files are uploaded, transcode will begin automatically and create the appropriate media item within the Brightcove Media system and transcoding will begin.

Create an XML Metadata File

Prepare a metadata file that describes the asset file to be ingested. This file should be named exactly the same as the media file, but the extension .xml. This metadata file is read as part of the ingest process. The information that can be provided in the metadata file includes:

*Required elements are in bold

Command

Description
Title Title of media asset
UniqueKey A globally unique alphanumeric string that identifies a media item

The language in which the asset is published (EN, FR, SP, etc.)

Example XML Asset Metadata File:

Using The Terminator as an example, the media file is named TerminatorMovie.mp4 and the metadata file is named TerminatorMovie.xml.

NOTE: While the system supports a number of metadata fields, only <title> and <uniquekey> are required. <foreignkey> can be any alphanumeric value. However, as best practice, we suggest using your system's Video ID or similar.

<rss version="1.0" xmlns:media="http://search.yahoo.com/mrss/" xmlns:uma="http://admin.Brightcovemedia.com/">
<item>
    <title>The Terminator</title>
    <description>A robotic assassin travels back in time.</description>
    <link> http://www.SomeCompany.com/TerminatorMovie</link>
    <copyright/>
    <pubDate>26 October 1984</pubDate>
    <uniquekey>12345_ABCDEF</uniquekey>
    <duration>1:47</duration>
    <type>video</type>
    <rating>PG</rating>
    <keywords>SCI-FI, ACTION, ADVENTURE, Arnold Schwarzenegger</keywords>
    <language>EN</language>
    <linkback>http://www.filmboxlive.com</linkback>
</item>

Ingest the Media File and its Metadata File

Once VODTM provides FTP upload capability. Nearly any FTP client that has the ability to use Active Transfer Mode can be used. We recommend the free FileZilla Client, which can be obtained online.

  1. Login to your assigned UM FTP server with a username and password

    1. Ensure your FTP client is set to use Active Transfer Mode
    2. Make an FTP connection to the Brightcove FTP Content Ingest server using the URL and credentials provided by Brightcove Client Services
    3. Within the FTP directory listing there will be a folder, corresponding to each of your catalogs, that is configured to use FTP Content Ingest

  2. Initiate an FTP transfer to the appropriate catalog for the media file and metadata file

    1. There will be a pause while the ingest software ensures that the media file has been transferred successfully

  3. When the files are 'transfer complete' (remaining unchanged for a short period) the metadata file will be parsed, the data extracted and then stored as metadata for the system media item

    1. Both files will be removed from the folder automatically

No further action is required from the user once the FTP transfer is completed.

Content Synchronization

Content Synchronization is the service that allows for the automated ingest of assets and metadata from an XML feed into the Brightcove ElementsTM platform. Our management platform polls every Content Synchronization feed regularly, about once per minute, in order to check whether the feed has changed since it was last ingested.

Process

We provide an XML node mapping function that relates an incoming feed from your system to properties in our media management platform allowing for the feed to be in any well-formed XML format. Therefore, if you have an existing feed with content, you don't have to modify it for us to ingest from it. A Brightcove Media Sales or Implementation Engineer can assist with configuring the appropriate Content Synchronization mapping.

XML and Customer Server Requirements

The requirements below are essential for a proper Content Synchronization implementation.

  • A publish date, unique ID and file location are required for each asset in the feed
  • Modified crossdomain.xml policies that allow necessary domains permitting Brightcove Media the ability to read the feed that will be ingested
  • Content needs to be publicly addressable (for origin server whitelisting, Brightcove Media IP ranges can be acquired form support@umedia.com)
  • Server must have last modified date turned on for the server the xml resides on, or it must be available via the xml's file header information
  • A feed URL that points to a specific XML file containing your content information Content server must accept byte range requests
XML Feed Structure

We don't require compliance with any standard XML feed structure. Primary requirements for feed ingest include:

  • The feed must contain multiple repeating nodes; one for each item
  • All content for each item must be contained within the node
  • The platform properties we require will be mapped based on the <item> node structure

NOTE: Metadata properties of an item may be mapped to an arbitrary XML node or attribute in the inbound feed.
