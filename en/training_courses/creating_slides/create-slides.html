Getting started

Introduction

This topic will walk you through creating a training course / doc tutorial together using our HTML slide template

The content for two will be identical, so you can create them in any order.

Tip: do the slide version first, as problems in the HTML will be more apparent there.

Files and folders

Set up your course

Here are the steps to set up your course:

  1. Create a folder for your course in the training_courses folder:

    BCL-developer-docs/en/training_courses/your-course-name
  2. Open BCL-developer-docs/en/templates/slide-template.html
  3. Save the file as index.html in the course folder you created
  4. You are ready to start editing the slides

Slide content

Special CSS classes

The following CSS classes are specific to these slides/docs:

  • slide - used with a div that will be the content for one slide. Any content not inside a <div class="slide"> will be on the screen for all slides (you probably don't want that)

  • slide-only - content that will only appear in the slides version

  • doc-only - content that will appear only in the doc version

  • bcls-highlight - highlight something to draw attention to it:

    .../data?<span class="bcls-highlight">accounts={account_id}</span>...
gives you:
.../data?accounts={account_id}...

  • BCL-aside use as you would in a doc

The start slide

The start slide is built into the template:

<!-- title slide -->
<div id="top" class="">
    <h1>Your title here</h1>
    <p class="doc-only">Intro paragraph for doc version only</p>
    <table class="slide-only bcls-float-right bcls-small">...<table>
</div>
  1. Change the title
  2. Change the intro content for the doc version only!
  3. Leave the table alone

End slide

An end slide is built into the template:

<!-- end slide - no content - for slides only -->
<div class="slide endSlide slide-only"></div>

Do not add any content to this slide

Tip: for on-demand recordings, you may want to break the course into multiple videos. You can just duplicate the end slide at the end of each recording section.

Section slides

Section slides provide title slides for sections:

<!-- section slide --> <div class="section slide" id="sectionID">
    <h2>Section header</h2>
</div>
  1. Change the section header
  2. Change the section id
  3. Don't add any other content

General content slides

<div id="slideID" class="slide">
    <h3>Slide heading</h3>
        other HTML....
</div>
  1. slide id optional but recommended for linking
  2. Start with an h3 to create the slide title
  3. Include any other HTML you like - be mindful of screen real-estate, preview and adjust, using doc-only to hide content in slides. If something doesn't look right, it's just CSS - contact Robert

Animations (slides in slides)

To have content slide in from the right, just create a slide within the slide:

<div class="slide">
    <p>Coming...</p>
    <ul><li>
        <div class="slide">
            ...soon (I appear on the next advance)
        </div>
    </li></ul>
</div>

Coming...

  • ...soon

Yes, you can have multiple levels of nested slides

Linking

Linking to docs- this content will live in two places so no relative links - use full URLs for all links (including scripts and stylesheets):

  • NO:
    href="../guides/overview.html"
  • YES:
    href="//docs.brightcove.com/en/video-cloud/di-api/guides/overview.html"

Linking to another slide

Use ids: href="#slideID"

The Back Button will work normally on internal or external links

Making the Doc

Create the doc

  1. Open BCL-developer-docs/en/templates/quick-start.html
  2. Save the file with the name you want and in the folder you want

Copy contents to doc

  1. Go back to your slides doc
  2. Look for this tag at the top of the content: 
    <div id="main" class="large-10 small-12 columns deck-container">
  3. Select the entire tag (in Sublime Text, place the cursor inside the tag and press CTRL-d)
  4. In your doc version, paste the copied code between these comments: 
    <!-- main content starts here -->

<!-- main content ends here -->
  5. To keep the content in sync, don't edit both docs! Always edit one, and copy the contents into the other as described here

General Tips

Tips

Do I have to be careful about which tags I use?
Only these:
  • h1 - only use for the deck/page title
  • h2 - only use for section titles
  • h3 - only use for slide titles
Can I add my own CSS?
Yes...but if it would be useful for other courses, ask Robert to add it to the general slide CSS instead
Can I add my own JavaScript?
Yes, you'll find a ready-made module at the end of the body...but if this is for slide manipulation rather than your content, ask Robert to add it to the template instead
