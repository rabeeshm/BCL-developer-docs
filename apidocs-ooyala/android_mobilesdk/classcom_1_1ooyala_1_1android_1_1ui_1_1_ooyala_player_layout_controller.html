Detailed Description

This LayoutController is a generic LayoutController that will work in most cases (regardless of the containing Layout type).

It uses basic controls and allows additional overlays to be added. Fullscreening is done by opening a full screen Dialog and filling it with a dynamically created OoyalaPlayerLayout. Because of this, playback will be suspended and subsequently resumed during this process. As a result, fullscreening is slower than if the OoyalaPlayerLayout is embeded directly in the Activity's base layout, that base layout is a FrameLayout, and the LayoutController used is FastOoyalaPlayerLayoutController.
